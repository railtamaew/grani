# Инструкция по тестированию XRay интеграции

## Предварительные требования

1. **Добавить библиотеки XRay-core:**
   - Скачать или собрать `.so` файлы для всех архитектур
   - Разместить в `android/app/src/main/jniLibs/`:
     - `arm64-v8a/libxray.so`
     - `armeabi-v7a/libxray.so`
     - `x86/libxray.so`
     - `x86_64/libxray.so`

2. **Реальное Android устройство** или эмулятор для тестирования

## Шаги тестирования

### 1. Подготовка библиотек

Если библиотеки еще не добавлены:

**Вариант A: Скачать готовые релизы**
```bash
# Перейти на https://github.com/XTLS/Xray-core/releases
# Скачать последнюю версию для Android
# Извлечь .so файлы в соответствующие директории
```

**Вариант B: Собрать через gomobile**
```bash
go install golang.org/x/mobile/cmd/gomobile@latest
gomobile init
git clone https://github.com/XTLS/Xray-core.git
cd Xray-core
gomobile bind -target=android -o xray.aar ./main
```

### 2. Сборка приложения

```bash
cd /opt/grani/mobile-app
flutter clean
flutter build apk --release
```

### 3. Установка на устройство

```bash
adb install build/app/outputs/flutter-apk/app-release.apk
```

### 4. Тестирование подключения

1. Запустить приложение на устройстве
2. Выбрать сервер, который поддерживает XRay протокол (VLESS/VMESS)
3. Выбрать протокол XRay (VLESS или VMESS)
4. Нажать кнопку подключения
5. Проверить логи:

```bash
adb logcat | grep -E "XrayCoreJni|GraniVpnService|VpnService"
```

### 5. Проверка подключения

- Проверить, что VPN интерфейс создан: `adb shell ip addr show tun0`
- Проверить, что трафик идет через VPN
- Проверить IP адрес: открыть браузер и проверить IP на whatismyip.com

## Ожидаемое поведение

1. **При успешном подключении:**
   - В логах должно быть: "XRay-core библиотека успешно загружена"
   - "XRay-core успешно запущен"
   - VPN интерфейс создан
   - IP адрес изменен на IP сервера

2. **При ошибках:**
   - Если библиотека не найдена: "Не удалось загрузить библиотеку libxray.so"
   - Если конфигурация невалидна: "Ошибка парсинга XRay конфигурации"
   - Если XRay не запустился: "Не удалось запустить XRay-core"

## Диагностика проблем

### Проблема: Библиотека не загружается

**Решение:**
- Убедитесь, что `.so` файлы находятся в правильных директориях
- Проверьте архитектуру устройства: `adb shell getprop ro.product.cpu.abi`
- Убедитесь, что файлы имеют правильные права доступа

### Проблема: XRay не запускается

**Решение:**
- Проверьте логи на наличие ошибок парсинга конфигурации
- Убедитесь, что конфигурация в правильном JSON формате
- Проверьте, что все обязательные поля присутствуют в конфигурации

### Проблема: VPN не подключается

**Решение:**
- Проверьте разрешения VPN в настройках Android
- Убедитесь, что сервер доступен и поддерживает выбранный протокол
- Проверьте логи на наличие ошибок подключения

## Примечания

- Тестирование требует реальных библиотек XRay-core
- JNI функции (`startXray`, `stopXray`, `isXrayRunning`) должны быть реализованы в нативной библиотеке
- Если используется готовая библиотека (например, `libv2ray`), может потребоваться адаптация JNI обертки под её API
