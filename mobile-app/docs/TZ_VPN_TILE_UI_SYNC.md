# ТЗ: Синхронизация UI приложения с состоянием VPN при использовании плитки быстрого доступа

**Дата:** 2026-02-24  
**Статус:** Реализовано (опрос при resumed + разовая синхронизация при открытии экрана подписки).

---

## 1. Описание проблемы

- При переключении VPN с плитки при открытом приложении статус в UI не обновлялся до сворачивания шторки или ухода в фон.
- При входе в приложение после подключения с плитки на экране могло отображаться «Отключено».

## 2. Реализованное решение

### 2.1 Опрос состояния при нахождении приложения на переднем плане

В `lib/main.dart` в `_AppLifecycleHandlerState`:

- При переходе в `AppLifecycleState.resumed` запускается периодический таймер (интервал 2 с), вызывающий `VpnService.syncConnectionStateWithNative()`.
- При переходе в `paused` или `detached` таймер отменяется.
- Таймер отменяется в `dispose()`.

Таким образом, при открытой шторке и переключении плитки состояние в приложении обновляется в течение не более 2 секунд.

### 2.2 Разовая синхронизация при открытии экрана подписки

В `lib/screens/trial_unified_screen.dart` в `initState` внутри `addPostFrameCallback`:

- Перед `refreshServers()` вызывается `vpnService.syncConnectionStateWithNative()` и ожидается завершение.
- Это обеспечивает актуальное состояние при первом показе экрана (в т.ч. после входа в приложение, когда VPN был включён/выключен с плитки).

## 3. Критерии приёмки (проверка)

1. Открыто приложение → опущена шторка → нажата плитка «Отключить»/«Подключить» → в течение не более нескольких секунд статус в приложении обновляется.
2. VPN подключен с плитки при закрытом/свёрнутом приложении → открытие приложения → на главном экране отображается «Подключено».
3. VPN отключен с плитки → открытие приложения → на экране отображается «Отключено».
4. Подключение/отключение кнопкой в приложении работает как раньше.

## 4. Дополнительно: падение tun2socks при отключении

В логах при отключении VPN наблюдалось:

```
FORTIFY: pthread_mutex_lock called on a destroyed mutex
Fatal signal 6 (SIGABRT) in tid ... (obile:tun2socks)
```

Рекомендуется отдельной задачей исправить порядок остановки tun2socks при stop VPN в нативном коде.
