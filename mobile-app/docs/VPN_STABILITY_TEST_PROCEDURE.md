# Тест стабильности VPN — процедура сбора логов

## Цель

Захватить логи при подключении/отключении в разных условиях (Wi‑Fi, мобильная сеть, разные операторы, разные протоколы), чтобы выявить и устранить нестабильности.

---

## 1. Подготовка

1. Установите приложение GraniVPN (собранный APK).
2. Убедитесь, что телефон подключён по USB и включена отладка по USB.
3. Очистите старые логи:
   ```bash
   adb logcat -c
   ```

---

## 2. Процедура теста (≈10–15 минут)

Выполняйте **каждый блок по порядку**. После блока можно сделать паузу 10–15 секунд.

### Блок A: Wi‑Fi, Reality
- Подключитесь к **Wi‑Fi**
- Откройте приложение, выберите протокол **Reality** (если есть)
- Подключитесь к VPN → дождитесь «Подключено»
- Подождите 30–60 с
- Отключите VPN
- Пауза 5–10 с

### Блок B: Wi‑Fi, VLESS
- Выберите протокол **VLESS**
- Подключитесь → «Подключено» → 30–60 с
- Отключите
- Пауза 5–10 с

### Блок C: Wi‑Fi, VMESS (если есть)
- Выберите **VMESS**
- Подключитесь → 30–60 с → отключите
- Пауза 5–10 с

### Блок D: Мобильная сеть (4G/5G), Reality
- Отключите Wi‑Fi, включите мобильные данные
- Подключитесь к VPN (Reality) → 30–60 с
- Отключите
- Пауза 5–10 с

### Блок E: Мобильная сеть, VLESS
- VLESS → подключение → 30–60 с → отключение
- Пауза 5–10 с

### Блок F: Переключение сети во время сессии
- Подключитесь к Wi‑Fi, подключите VPN (Reality)
- Через 20–30 с выключите Wi‑Fi (VPN должен перейти на мобильную сеть)
- Подождите 30 с
- Включите Wi‑Fi обратно
- Подождите 20 с
- Отключите VPN

### Блок G: Быстрое reconnect (опционально)
- Wi‑Fi, подключите VPN (Reality)
- Через 10 с отключите
- **Сразу** подключите снова (reconnect из кэша)
- Через 20 с отключите

---

## 3. Захват логов

**Перед тестом:** `adb logcat -c`  
**После теста** (выполнив все блоки A–G):

### Вариант 1: Полный дамп (рекомендуется)

```bash
adb logcat -d > vpn_test_$(date +%Y%m%d_%H%M%S).logcat
```

Файл сохранится в текущей папке. Его можно передать на анализ.

### Вариант 2: С фильтром по приложению (меньше мусора)

```bash
adb logcat -d "*:V" | grep -E "VpnService|Tun2SocksProc|XrayNativeWrapper|ConnectionLogger|flutter|granivpn" > vpn_test_filtered.txt
```

### Вариант 3: Полный JSON (если используете Android Studio Device Explorer)

1. Device Explorer → View Tool Window → Logcat
2. Run тест, затем Save Logcat to File
3. Сохраните как `.logcat` или `.json`

---

## 4. Что передать на анализ

- Файл логов: `vpn_test_YYYYMMDD_HHMMSS.logcat` (или `.txt`)
- Краткое описание: какие блоки прошли, где было нестабильно (долгие подключения, подвисания при отключении, ошибки в UI)

---

## 5. Что оценивается в логах

| Проблема | Паттерн в логах |
|----------|-----------------|
| Native crash tun2socks | `pthread_mutex_lock`, `Fatal signal 6` |
| Reconnect без трафика | `tcp_bind_to_netif failed` |
| Переполнение буфера TUN | `no space left` |
| Таймаут отправки логов | `ConnectionLogger: Ошибка`, `connection timeout` |
| Bind timeout tun2socks | `Tun2SocksProcessService bind timeout` |
| Медленное подключение | `total_ms` > 8000 |
| Медленное отключение | долгий disconnect flow |
| Ошибки по протоколам | `connection_error`, `protocol_error` |

После анализа логов будут предложены конкретные изменения в коде.
