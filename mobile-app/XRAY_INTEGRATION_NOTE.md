# Важное замечание по интеграции libXray

## Обнаруженная проблема

Библиотека libXray экспортирует функции с префиксом `Java_libXray_`, а не `Java_com_granivpn_mobile_XrayCoreJni_`.

Это означает, что текущая JNI обертка `XrayCoreJni.kt` не будет работать напрямую с библиотекой libXray.

## Решения

### Вариант 1: Использовать готовые классы из libXray AAR (рекомендуется)

libXray предоставляет готовые Java/Kotlin классы, которые можно использовать напрямую:

1. Добавить AAR как зависимость в `build.gradle`:
```gradle
dependencies {
    implementation files('libs/libXray.aar')
}
```

2. Использовать классы libXray вместо прямых JNI вызовов

### Вариант 2: Адаптировать JNI обертку

Изменить имена функций в `XrayCoreJni.kt` под реальные функции libXray:
- `Java_libXray_LibXray__1init` - инициализация
- `Java_libXray_LibXray_convertShareLinksToXrayJson` - конвертация конфигурации
- И другие функции

### Вариант 3: Создать собственную обертку

Создать промежуточный слой, который будет вызывать функции libXray через правильные имена JNI.

## Текущее состояние

- ✅ Библиотеки собраны и добавлены в проект
- ✅ Структура директорий готова
- ⚠️ Требуется адаптация JNI обертки или использование готовых классов libXray

## Рекомендация

Использовать готовые классы из libXray AAR вместо прямых JNI вызовов, так как это:
- Более надежно
- Уже протестировано
- Имеет правильную структуру API
