# План тестирования SingBoxWrapper

## Обзор

Этот документ описывает план тестирования для `SingBoxWrapper` - обертки для sing-box (libbox) библиотеки, которая обеспечивает работу VPN протоколов Xray (VLESS, VMESS, REALITY).

## Типы тестов

### 1. Unit тесты (✅ Созданы)

**Файл:** `android/app/src/test/kotlin/com/granivpn/mobile/SingBoxWrapperTest.kt`

**Покрытие:**
- ✅ Создание SingBoxWrapper
- ✅ Запуск VPN с валидными конфигурациями (VLESS, VMESS, REALITY)
- ✅ Обработка невалидных конфигураций
- ✅ Остановка VPN
- ✅ Проверка статуса (isRunning)
- ✅ Получение статистики трафика
- ✅ Преобразование XrayConfig в sing-box JSON

**Запуск:**
```bash
cd /opt/grani/mobile-app/android
./gradlew test
```

**Статус:** Базовые тесты созданы, требуют доработки для полной интеграции с libbox

---

### 2. Интеграционные тесты

**Цель:** Проверить работу SingBoxWrapper с реальной библиотекой libbox

**Сценарии:**

#### 2.1 Тест создания TUN интерфейса
- **Цель:** Проверить, что TUN интерфейс создается корректно
- **Шаги:**
  1. Создать SingBoxPlatform
  2. Вызвать createTunInterface()
  3. Проверить, что ParcelFileDescriptor не null
  4. Проверить, что FD валиден
- **Ожидаемый результат:** TUN интерфейс создан успешно

#### 2.2 Тест преобразования конфигурации
- **Цель:** Проверить корректность преобразования XrayConfig в sing-box JSON
- **Шаги:**
  1. Создать XrayConfig для каждого протокола (VLESS, VMESS, REALITY)
  2. Вызвать toSingBoxJsonConfig()
  3. Валидировать JSON через Libbox.checkConfig()
- **Ожидаемый результат:** Все конфигурации валидны

#### 2.3 Тест запуска VPN для каждого протокола
- **Цель:** Проверить запуск VPN для каждого поддерживаемого протокола
- **Протоколы:**
  - VLESS + TLS
  - VLESS + REALITY
  - VMESS + TLS
  - VMESS + WebSocket
- **Шаги:**
  1. Создать валидную конфигурацию
  2. Вызвать startVpn()
  3. Проверить, что CommandServer создан
  4. Проверить, что isRunning() возвращает true
- **Ожидаемый результат:** VPN запускается для всех протоколов

#### 2.4 Тест остановки VPN
- **Цель:** Проверить корректную остановку VPN
- **Шаги:**
  1. Запустить VPN
  2. Вызвать stopVpn()
  3. Проверить, что CommandServer закрыт
  4. Проверить, что TUN интерфейс закрыт
  5. Проверить, что isRunning() возвращает false
- **Ожидаемый результат:** VPN полностью остановлен, ресурсы освобождены

#### 2.5 Тест обработки ошибок
- **Цель:** Проверить обработку различных ошибок
- **Сценарии:**
  - Невалидная конфигурация
  - Ошибка создания TUN интерфейса
  - Ошибка создания CommandServer
  - Ошибка запуска сервиса
- **Ожидаемый результат:** Все ошибки обрабатываются корректно, ресурсы освобождаются

**Файл для создания:** `android/app/src/androidTest/kotlin/com/granivpn/mobile/SingBoxWrapperIntegrationTest.kt`

**Запуск:**
```bash
cd /opt/grani/mobile-app/android
./gradlew connectedAndroidTest
```

### 3. Системные тесты (на реальном устройстве)

**Цель:** Проверить работу VPN на реальном Android устройстве

**Сценарии:**

#### 3.1 Тест подключения к реальному серверу
- **Цель:** Проверить реальное подключение через VPN
- **Шаги:**
  1. Установить APK на устройство
  2. Запустить приложение
  3. Выбрать сервер с поддержкой Xray
  4. Выбрать протокол (VLESS/VMESS/REALITY)
  5. Подключиться к VPN
  6. Проверить, что IP адрес изменился
  7. Проверить доступность интернета
- **Ожидаемый результат:** VPN работает, трафик идет через туннель

#### 3.2 Тест всех протоколов
- **Цель:** Проверить работу всех поддерживаемых протоколов
- **Протоколы:**
  - VLESS + TLS
  - VLESS + REALITY
  - VMESS + TLS
  - VMESS + WebSocket
- **Ожидаемый результат:** Все протоколы работают корректно

#### 3.3 Тест переподключения
- **Цель:** Проверить стабильность при переподключении
- **Шаги:**
  1. Подключиться к VPN
  2. Отключиться
  3. Подключиться снова
  4. Повторить 5-10 раз
- **Ожидаемый результат:** Переподключение работает стабильно

#### 3.4 Тест работы при переключении сетей
- **Цель:** Проверить работу при смене сети (Wi-Fi ↔ Mobile)
- **Шаги:**
  1. Подключиться к VPN через Wi-Fi
  2. Переключиться на мобильную сеть
  3. Проверить, что VPN продолжает работать
- **Ожидаемый результат:** VPN работает при смене сети

#### 3.5 Тест статистики трафика
- **Цель:** Проверить сбор статистики трафика
- **Шаги:**
  1. Подключиться к VPN
  2. Передать некоторый трафик
  3. Получить статистику через getTrafficStats()
  4. Проверить, что статистика обновляется
- **Ожидаемый результат:** Статистика собирается корректно

**Запуск:**
```bash
# Установить APK на устройство
adb install /opt/grani/mobile-app/build/app/outputs/apk/debug/app-debug.apk

# Запустить логирование
adb logcat | grep -E "SingBoxWrapper|GraniVpnService|libbox"
```

### 4. Тесты производительности

**Цель:** Проверить производительность VPN подключения

**Метрики:**
- Время запуска VPN (должно быть < 2 секунд)
- Задержка (ping) через VPN (должна быть < 100ms для локальных серверов)
- Пропускная способность (должна быть > 10 Mbps)
- Использование памяти (должно быть < 100 MB)
- Использование CPU (должно быть < 20% при простоя)

**Инструменты:**
- Android Profiler
- `adb shell dumpsys meminfo`
- `adb shell top`

## Примечания

1. **Моки libbox:** Так как libbox - это нативная библиотека, для unit тестов потребуются моки. Можно использовать MockK или создать интерфейсы-обертки.

2. **Тестовые серверы:** Для системных тестов нужны тестовые VPN серверы с поддержкой всех протоколов.

3. **Автоматизация:** Рекомендуется настроить CI/CD для автоматического запуска тестов при каждом коммите.
