## Важно: Имя библиотеки

libXray создает библиотеку с именем `libgojni.so`, но для нашего проекта она скопирована как `libxray.so`. 

**Текущая реализация:**
- JNI обертка (`XrayCoreJni.kt`) пытается загрузить `libxray.so`
- Библиотека должна экспортировать функции через JNI

**Если библиотека не загружается:**
- Проверьте, что libXray экспортирует функции с правильными именами JNI
- Возможно, потребуется адаптация JNI обертки под API libXray
- Или использование готовых Java/Kotlin классов из AAR вместо прямых JNI вызовов
