# Рекомендации по тестированию VPN приложения

## 🎯 Приоритет тестирования протоколов

### 1. **WireGuard** (ПЕРВЫЙ ПРИОРИТЕТ) ⭐
**Почему первым:**
- Самый стабильный и быстрый протокол
- Наименьшая вероятность ошибок
- Простая конфигурация
- Хорошая поддержка на Android

**Что тестировать:**
- ✅ Подключение к серверу
- ✅ Получение конфигурации через `/vpn/connect`
- ✅ Применение конфигурации
- ✅ Проверка подключения (IP, DNS, трафик)
- ✅ Отключение
- ✅ Повторное подключение
- ✅ Обработка ошибки "Устройство уже подключено"

**Ожидаемое поведение:**
- Подключение должно установиться за 5-10 секунд
- IP адрес должен измениться
- DNS должен работать
- Трафик должен проходить через VPN

## 📋 Чек-лист тестирования

### Перед началом тестирования:
- [ ] Убедитесь, что приложение установлено с нуля (чистая установка)
- [ ] Проверьте, что устройство подключено к интернету
- [ ] Убедитесь, что сервер доступен и работает
- [ ] Проверьте, что у вас есть валидный токен авторизации

### Базовое тестирование (для каждого протокола):

#### 1. Тест подключения
- [ ] Нажмите кнопку подключения
- [ ] Проверьте прогресс подключения (должен показывать этапы)
- [ ] Убедитесь, что подключение завершилось успешно
- [ ] Проверьте, что статус изменился на "Подключено"

#### 2. Тест получения конфигурации
- [ ] Проверьте логи: для XRay должен использоваться `/vpn/xray/create-client`
- [ ] Проверьте логи: для WireGuard/GRANIWG/OpenVPN должен использоваться `/vpn/connect`
- [ ] Убедитесь, что конфигурация получена и сохранена

#### 3. Тест проверки подключения
- [ ] Проверьте, что IP адрес изменился
- [ ] Проверьте, что DNS работает (откройте браузер)
- [ ] Проверьте статистику трафика (должен быть трафик)

#### 4. Тест отключения
- [ ] Нажмите кнопку отключения
- [ ] Убедитесь, что VPN отключен
- [ ] Проверьте, что IP адрес вернулся к исходному

#### 5. Тест повторного подключения
- [ ] Подключитесь снова
- [ ] Убедитесь, что подключение работает
- [ ] Проверьте, что нет ошибок

#### 6. Тест обработки ошибки "Устройство уже подключено"
- [ ] Если возникает ошибка, проверьте логи:
  - [ ] Должно быть сообщение "⚠️ Устройство уже подключено на сервере, отключаем..."
  - [ ] Должно быть сообщение "Устройство отключено, повторяем попытку подключения..."
  - [ ] Должно быть сообщение "✅ Повторная попытка успешна"
- [ ] Подключение должно завершиться успешно после повторной попытки

## 🔍 Детальное тестирование

### Тест 1: WireGuard (Базовый)
1. Выберите протокол WireGuard
2. Нажмите "Подключиться"
3. **Проверьте логи:**
   ```
   VpnService.connect: ✅ Конфигурация получена
   VpnService.connect: XRay протокол - используем _connectInternal() для получения конфигурации
   ```
   ❌ НЕ должно быть для WireGuard!
4. **Проверьте логи:**
   ```
   VpnService._connectWireGuard: Отправляем запрос на /vpn/connect
   ```
5. Убедитесь, что подключение успешно

### Тест 2: XRay Reality (Критический)
1. Выберите протокол XRay Reality
2. Нажмите "Подключиться"
3. **Проверьте логи:**
   ```
   VpnService.connect: XRay протокол - конфигурация будет получена в _connectXray()
   VpnService.connect: XRay протокол - используем _connectInternal() для получения конфигурации
   ```
4. **Проверьте логи:**
   ```
   VpnService._connectXray: Отправляем запрос на /vpn/xray/create-client
   ```
   ✅ Должен быть `/vpn/xray/create-client`, НЕ `/vpn/connect`!
5. Убедитесь, что подключение успешно

### Тест 3: Обработка ошибки "Устройство уже подключено"
1. Подключитесь к VPN
2. Не отключайтесь, попробуйте подключиться снова
3. **Проверьте логи:**
   ```
   VpnService.connect: DioException поймана, statusCode: 400
   VpnService.connect: errorMessage: Устройство уже подключено
   VpnService.connect: ⚠️ Устройство уже подключено на сервере, отключаем...
   VpnService._forceDisconnectOnServer: Принудительное отключение устройства на сервере
   VpnService.connect: Устройство отключено, повторяем попытку подключения...
   VpnService.connect: ✅ Повторная попытка успешна
   ```
4. Убедитесь, что подключение завершилось успешно

### Тест 4: Синхронизация состояния при старте
1. Закройте приложение во время подключения
2. Откройте приложение снова
3. **Проверьте логи:**
   ```
   VpnService._syncConnectionState: Проверка состояния подключения на сервере...
   ```
4. Убедитесь, что состояние синхронизировано

## 🐛 Известные проблемы и их решения

### Проблема: "Устройство уже подключено"
**Решение:** Обработка добавлена, но если проблема сохраняется:
1. Проверьте логи на наличие сообщения "⚠️ Устройство уже подключено"
2. Если сообщения нет - обработка не срабатывает, проверьте код
3. Если сообщение есть, но подключение не работает - проверьте сервер

### Проблема: XRay использует неправильный эндпоинт
**Решение:** Исправлено в коде. Проверьте логи:
- Для XRay должен быть `/vpn/xray/create-client`
- НЕ должно быть `/vpn/connect` для XRay

### Проблема: Конфигурация не применяется
**Решение:** 
1. Проверьте, что конфигурация получена
2. Проверьте, что конфигурация валидна для протокола
3. Проверьте логи применения конфигурации
