### Nginx access.log

Другие клиенты (94.180.243.40, 85.26.164.58) успешно получают 200:

- `/api/auth/google/callback` (косвенно, после успешной авторизации)

### Вывод
**Запросы с OnePlus не доходят до сервера.** connection timeout возникает на стороне клиента до установления TCP/TLS. Nginx не получает запрос и не пишет его в access.log.

## Возможные причины

1. **Сеть оператора** — блокировка, DPI или особенности маршрутизации до api.granilink.com и 159.223.199.122.
2. **IPv6** — приоритет IPv6 (Happy Eyeballs), нерабочий IPv6-путь, таймаут, fallback на IPv4 слишком поздно.
3. **TLS** — несовместимость (в т.ч. «bad key share»), но тогда обычно соединение до Nginx доходит и ошибка есть в error.log; за 12:48–12:53 UTC в error.log записей нет.
4. **Firewall на сервере** — блокировка отдельных IP/подсетей (проверить iptables, ufw).

## Рекомендации

1. Проверить доступность API с устройства: в браузере открыть `https://api.granilink.com/api/vpn/bootstrap`.
2. Пробовать другую сеть: Wi‑Fi дома, другой оператор, мобильный интернет.
3. Отключить IPv6 на устройстве или в приложении (если есть возможность) и проверить снова.
4. Увеличить `connectTimeout` в приложении (до 15–20 с) — может помочь при медленном пути.
5. Проверить, блокирует ли firewall IP устройства: `sudo iptables -L -n`, `sudo ufw status`.
