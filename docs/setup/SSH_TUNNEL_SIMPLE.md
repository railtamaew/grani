# Простая настройка SSH туннеля для ADB

## Важно!

На сервере нет пользователя `rail`, есть только `root`. Используйте `root` для SSH туннеля.

## Шаг 1: На Windows компьютере

### 1. Настройте ADB (если еще не сделали)

Откройте PowerShell:

```powershell
# Перейдите в папку с ADB из Android Studio
cd C:\Users\railt\AppData\Local\Android\Sdk\platform-tools

# Проверьте подключение устройства
.\adb.exe devices

# Включите TCP/IP режим
.\adb.exe tcpip 5555

# Подключитесь к устройству локально
.\adb.exe connect 172.16.105.33:5555

# Проверьте - должно показать устройство
.\adb.exe devices
```

### 2. Создайте SSH туннель

В **НОВОМ окне PowerShell** выполните:

```powershell
ssh -R 5555:localhost:5555 root@159.223.199.122 -N
```

**Важно:**
- Используйте `root` вместо `rail`
- Если SSH ключ настроен - пароль не потребуется
- Если ключ не настроен - введите пароль пользователя `root` на сервере
- **Оставьте это окно открытым** - туннель должен работать

## Шаг 2: На сервере (после создания туннеля)

После того как SSH туннель будет создан, на сервере выполните:

```bash
export PATH=$PATH:/opt/android-sdk/platform-tools
adb connect localhost:5555
adb devices
```

Или просто запустите:

```bash
/opt/grani/dev_run.sh
```

## Проверка

После создания туннеля проверьте на сервере:

```bash
# Проверка порта
ss -tln | grep 5555

# Подключение к устройству
adb connect localhost:5555
adb devices
```

## Если SSH ключ не настроен

Если при подключении запрашивается пароль, введите пароль пользователя `root` на сервере.

Если не знаете пароль, но у вас есть доступ через Cursor, значит SSH ключ уже настроен и пароль не потребуется.
