# Запуск ADB через SSH туннель

## Текущая ситуация

- Пользователь: rail
- Сервер: 159.223.199.122
- Устройство: 172.16.105.33

## Шаг 1: На вашем Windows компьютере

### 1. Настройте ADB (если еще не сделали)

Откройте PowerShell и выполните:

```powershell
# Перейдите в папку с ADB из Android Studio
cd C:\Users\railt\AppData\Local\Android\Sdk\platform-tools

# Проверьте подключение устройства
.\adb.exe devices

# Включите TCP/IP режим
.\adb.exe tcpip 5555

# Подключитесь к устройству локально
.\adb.exe connect 172.16.105.33:5555

# Проверьте
.\adb.exe devices
```

Должно показать: `172.16.105.33:5555    device`

### 2. Создайте SSH туннель

В **НОВОМ окне PowerShell** выполните:

```powershell
ssh -R 5555:localhost:5555 root@159.223.199.122 -N
```

Эта команда:
- Создаст туннель от порта 5555 на сервере к порту 5555 на вашем компьютере
- `-N` означает, что не нужно выполнять команды, только туннель
- **Оставьте это окно открытым** - туннель должен работать

Если потребуется пароль, введите его.

## Шаг 2: На сервере (после создания туннеля)

После того как SSH туннель будет создан, на сервере выполните:

```bash
export PATH=$PATH:/opt/android-sdk/platform-tools
adb connect localhost:5555
adb devices
```

Или просто запустите:

```bash
/opt/grani/dev_run.sh
```

## Проверка

После создания туннеля проверьте на сервере:

```bash
# Проверка порта
netstat -tln | grep 5555

# Подключение к устройству
adb connect localhost:5555
adb devices
```

## Если туннель не работает

1. **Проверьте SSH подключение:**
   ```powershell
   ssh root@159.223.199.122
   ```

2. **Проверьте, что ADB работает локально:**
   ```powershell
   .\adb.exe devices
   ```

3. **Попробуйте с verbose режимом:**
   ```powershell
   ssh -v -R 5555:localhost:5555 root@159.223.199.122 -N
   ```

4. **Проверьте firewall на Windows** - может блокировать порт 5555

## Альтернатива: Прямое подключение (если в одной сети)

Если устройство и сервер в одной сети, можно подключиться напрямую:

```bash
/opt/grani/connect_remote_device.sh 172.16.105.33
```

Но обычно нужен SSH туннель, так как устройство в локальной сети пользователя.
