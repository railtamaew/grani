# Лабораторная оценка сценариев обхода блокировок

Пошаговое руководство для **первого пункта** [порядка действий](VPN_BYPASS_TESTING_AND_INTEGRATION.md#5-порядок-действий-перед-выбором-реализации): провести лабораторную оценку одного–двух приоритетных сценариев (рекомендуется 6 + 4 или 6 + 1).

Цель: до полной реализации убедиться, что трафик кандидата не попадает под типичные правила блокировки (SNI, отпечаток протокола) и при необходимости сравнить с эталонным сервисом (JA3, размеры пакетов).

---

## 1. Подготовка окружения

### 1.1. Захват трафика

- **tcpdump** (часто уже установлен):
  ```bash
  # Захват 60 сек на интерфейсе (подставьте свой: eth0, en0, tun0 и т.д.)
  sudo tcpdump -i any -w /tmp/capture.pcap -s 0 host not 127.0.0.1
  ```
- **Wireshark/tshark** (для разбора pcap и JA3):
  ```bash
  # Установка (пример для Ubuntu/Debian)
  sudo apt install tshark
  # Список TLS Client Hello и JA3 (если поддерживается версией)
  tshark -r capture.pcap -Y "tls.handshake.type == 1" -T fields -e tls.handshake.ja3 2>/dev/null || true
  ```

### 1.2. Скрипт из репозитория

Из корня проекта:

```bash
# Захват 30 сек в файл (требуются права на захват пакетов)
python backend/scripts/bypass_lab_capture_traffic.py --duration 30 --out /tmp/bypass_lab.pcap

# Анализ уже записанного pcap
python backend/scripts/bypass_lab_capture_traffic.py --analyze /tmp/bypass_lab.pcap
```

Скрипт выводит краткую сводку (соединения, порты, размеры первых пакетов) и подсказки по дальнейшему анализу JA3/SNI.

## 2. Сценарии оценки

### Сценарий A: Сравнение текущего стека с «эталоном»

1. Записать трафик **текущего приложения** (WireGuard или XRay) при активном туннеле: браузер или приложение ходит в интернет через VPN.
2. Записать трафик **эталонного сервиса** (например, один и тот же сайт или API в браузере без VPN — тот же Chrome/Firefox).
3. Сравнить:
   - **JA3/JA4** (TLS Client Hello): совпадает ли отпечаток с эталоном или явно другой (типичный для библиотек VPN).
   - **Размеры первых пакетов** (первые 5–10 пакетов по направлению клиент→сервер): похожи ли на эталон или стабильно другие.
   - **SNI**: виден ли один и тот же домен или «подозрительный» домен/IP.

Инструменты для JA3 (если tshark не выводит): [ja3](https://github.com/salesforce/ja3), [PyJA3](https://github.com/salesforce/ja3), или онлайн-анализ pcap (осторожно с конфиденциальностью).

### Сценарий B: Проверка кандидата «Boring HTTPS» (сценарий 4)

1. Реализовать минимальный PoC: клиент открывает одно HTTPS-соединение к выбранному «прикрывающему» домену и в теле POST передаёт зашифрованный фрагмент туннеля (или заглушку).
2. Записать трафик этого клиента.
3. Проверить:
   - В SNI виден только выбранный домен (например, легитимный CDN/облако).
   - JA3 совпадает с типичным для браузера или мобильного приложения (Chrome/Android и т.д.), а не с отпечатком известных VPN-библиотек.
   - Нет ли лишних соединений к «подозрительным» портам или доменам.

### Сценарий C: Проверка Bootstrap (сценарий 6)

1. Реализовать цепочку: DoH → получение URL конфига → HTTPS-запрос к этому URL (прикрывающий домен).
2. Записать трафик.
3. Убедиться, что ни на одном этапе не фигурирует явный «vpn»-домен в SNI; запросы выглядят как обычные к CDN/облаку.

## 3. Эмуляция блокировки (опционально)

Чтобы проверить «не попадает под правила», можно поставить между клиентом и сетью прокси, который:

- **Блокирует по SNI:** при появлении в TLS Client Hello заданного SNI — закрывает соединение или не пропускает пакеты.
- **Логирует JA3/размеры:** записывает в лог для последующего сравнения.

Примеры инструментов:

- **mitmproxy** (прозрачный прокси): можно писать addon на Python, который инспектирует SNI и разрывает соединение при совпадении с блок-листом. Для TLS нужно, чтобы клиент доверял CA mitmproxy (подходит для контролируемого теста).
- **Кастомный прокси** на Python (asyncio + как минимум чтение первого пакета TLS для извлечения SNI): без расшифровки можно парсить Client Hello и по SNI решать, пропускать или дропать.

После настройки такого прокси запускать клиент через него и убеждаться, что трафик кандидата (с «прикрывающим» SNI) не дропается при правилах «блокировать только vpn.example».
