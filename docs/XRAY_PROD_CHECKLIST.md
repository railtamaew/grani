# Xray Production Checklist

## Цели
- Подтвердить стабильную работу VLESS/REALITY/VMESS.
- Проверить наблюдаемость и корректность health_status.
- Зафиксировать регресс-сценарии перед релизом.

## Предусловия
- Серверы синхронизированы по Xray конфигу.
- Последний APK установлен на тестовом устройстве.
- Устройство подключается через нужный протокол.

## REALITY
- Подключение: установить протокол `Xray Reality` и подключиться.
- Сервер: нет `failed to read client hello` в `error.log`.
- Клиент: в логах есть `proxy/vless/outbound` и трафик TCP/UDP.
- Трафик: открыть 2-3 HTTPS сайта + проверка UDP (например, видеосервис).

## VMESS
- Подключение: установить протокол `Xray VMESS` и подключиться.
- Сервер: есть `proxy/vmess/inbound` и `proxy/freedom` в логах.
- Клиент: нет `unsupported encryption: auto`.
- Трафик: открыть 2-3 HTTPS сайта + DNS резолв.

## VLESS
- Подключение: установить `Xray VLESS` и подключиться.
- Сервер: активные соединения и трафик видны в access/error логах.

## Health Status
- После успешного подключения `health_status` в списке серверов = `healthy`.
- После серии ошибок = `warning`.
- Проверить, что статус обновляется в течение 1-2 минут.

## Регресс-сценарии
- Переключение протоколов: VLESS → REALITY → VMESS.
- Перезапуск приложения без выхода: состояние корректно.
- Отключение сети и повторное подключение.
- 15–30 минут стабильной работы без разрывов.

## Диагностика
- Серверные логи Xray: `error.log`, `access.log`.
- Клиентские логи: `XrayNativeWrapper`, `VpnService`.
- Проверка `health_status` через API `/api/vpn/servers`.

## Критерии успешности
- Подключение стабильно на всех трех протоколах.
- Реальный трафик проходит через сервер.
- Логи без системных ошибок Reality/VMESS.
