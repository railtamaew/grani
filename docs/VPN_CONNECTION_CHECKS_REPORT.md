## 5. Выводы и рекомендации

### С сервера API работает
- `api.granilink.com` отвечает быстро
- Backend и Nginx функционируют

### Возможные причины timeout на устройстве пользователя

1. **Сеть** — блокировка или фильтрация api.granilink.com и/или 159.223.199.122 у оператора/провайдера.

2. **IP fallback (159.223.199.122)** — SSL exit 60: при подключении по IP и проверке сертификата клиент может отклонять соединение. Проверить в мобильном приложении обработку SSL при запросах по IP.

3. **TLS "bad key share"** — часть клиентов получает сбой на этапе handshake. Рассмотреть изменение `ssl_ciphers` или `ssl_protocols` в Nginx.

4. **Rate limiting** — при большом количестве запросов с одного IP возможна 503; маловероятно как причина connection timeout, но стоит учитывать.

### Рекомендации
- Проверить доступность api.granilink.com с устройства пользователя (браузер, другая сеть).
- При использовании 159.223.199.122 — проверить и при необходимости смягчить проверку SSL в Dio/HttpClient.
- Отслеживать `bad key share` в Nginx — при повторениях изменить конфигурацию SSL.
