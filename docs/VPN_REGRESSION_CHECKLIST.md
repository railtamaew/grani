# Регрессия VPN после единого Xray-пути (без отдельного процесса `:xray`) и ожиданий готовности

Чеклист для ручной проверки на реальном устройстве (Android). Цель: connect / reconnect / logout и смена сети стабильны после удаления отдельного процесса Xray и ожидания готовности нативного ядра перед tun2socks.

## Подготовка

- Сборка с актуальным нативным слоем (TUN, tun2socks, Xray в процессе `VpnService`).
- Аккаунт с подпиской или trial, несколько серверов и протоколов Xray (VLESS, VMESS, REALITY).

## Сценарии

1. **Холодное подключение** — убить приложение, открыть, подключиться. Ожидание: прогресс до «подключено», трафик через VPN.

2. **Повторное подключение (warm)** — отключить VPN, снова подключиться к тому же серверу. Ожидание: без зависаний на «запуск протокола» / «проверка трафика».

3. **Logout при активном VPN** — включить VPN, выйти из аккаунта. Ожидание: VPN останавливается, нет краша, UI согласован (`vpnUiSessionState`, `isVpnSessionPotentiallyActive`).

4. **Смена сети Wi‑Fi и мобильные данные** — при активном VPN переключить сеть. Ожидание: восстановление или предсказуемый disconnect и ручной reconnect без перезапуска приложения.

5. **Смена сети после удаления клиента на сервере** — подключиться, удалить клиента или сменить порт на ноде (по вашему процессу), переподключиться. Ожидание: не держать устаревший кэш с неверным портом (валидация по `xray_expected_ports` из bootstrap и fallback из констант бэкенда).

6. **Quick tile (если включён)** — включение/выключение из шторки при приложении в фоне.

## Логи

- Android: `adb logcat` — `VpnService`, Xray, tun2socks, ошибки bind и timeout.
- API: `POST /api/vpn/session/prepare` — см. [OPS_SESSION_PREPARE_METRICS.md](./OPS_SESSION_PREPARE_METRICS.md).
