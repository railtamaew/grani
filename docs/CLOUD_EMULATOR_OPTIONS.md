# Облачные эмуляторы и альтернативы для тестирования

## Что такое облачный эмулятор?

Облачный эмулятор - это Android эмулятор, работающий на удаленном сервере, к которому можно подключиться через веб-интерфейс или API. Не требует установки и настройки на локальной машине.

## Варианты облачных эмуляторов

### 1. Firebase Test Lab (Google) - Рекомендуется

**Преимущества:**
- ✅ Бесплатный квотный лимит (до 10 тестов в день)
- ✅ Веб-интерфейс для просмотра результатов
- ✅ CLI для автоматизации
- ✅ Множество устройств и версий Android
- ✅ Автоматические скриншоты и видео

**Использование:**
```bash
# Установка Firebase CLI
npm install -g firebase-tools

# Вход в аккаунт Google
firebase login

# Запуск теста
gcloud firebase test android run \
  --app app-debug.apk \
  --device model=Pixel4,version=30 \
  --timeout 5m
```

**Ссылки:**
- https://firebase.google.com/docs/test-lab
- https://console.firebase.google.com/

### 2. AWS Device Farm

**Преимущества:**
- ✅ Поддержка реальных устройств
- ✅ Интеграция с CI/CD
- ✅ Автоматические тесты

**Недостатки:**
- ❌ Платный сервис
- ❌ Сложнее настройка

### 3. BrowserStack / Sauce Labs

**Преимущества:**
- ✅ Простой веб-интерфейс
- ✅ Множество устройств

**Недостатки:**
- ❌ Платный сервис
- ❌ Ограниченная автоматизация

## Альтернативные решения

### 1. Физическое устройство через USB (Самый простой)

**Преимущества:**
- ✅ Не требует облачных сервисов
- ✅ Реальное устройство (лучше для VPN тестирования)
- ✅ Уже есть скрипты в проекте (`INSTALL_APK.sh`)

**Использование:**
```bash
# Подключите устройство через USB
# Включите отладку по USB в настройках разработчика

# Проверка подключения
adb devices

# Установка и запуск
/opt/grani/INSTALL_APK.sh
```

### 2. Исправить локальный эмулятор

**Проблема:** Эмулятор падает при запуске (core dumped)

**Возможные решения:**
1. Пересоздать AVD с другими параметрами
2. Использовать более старую версию эмулятора
3. Проверить системные требования (KVM, виртуализация)
4. Использовать headless режим без VNC

### 3. Docker контейнер с эмулятором

**Преимущества:**
- ✅ Изолированная среда
- ✅ Легко пересоздать при проблемах

**Недостатки:**
- ❌ Требует Docker
- ❌ Может быть медленнее

## Рекомендация для текущего проекта

**Лучший вариант:** Использовать физическое Android устройство

**Причины:**
1. Уже есть готовые скрипты (`INSTALL_APK.sh`, `start_logcat_monitor.sh`)
2. Реальное устройство лучше для тестирования VPN
3. Не требует облачных сервисов
4. Можно давать разрешения VPN вручную на устройстве

**Если нет физического устройства:**
1. Попробовать исправить локальный эмулятор
2. Использовать Firebase Test Lab для автоматических тестов
3. Рассмотреть Docker контейнер с эмулятором

## Быстрый старт с физическим устройством

```bash
# 1. Подключите устройство через USB
# 2. Включите отладку по USB в настройках разработчика

# 3. Проверка
adb devices

# 4. Установка APK
/opt/grani/INSTALL_APK.sh

# 5. Мониторинг логов
/opt/grani/scripts/start_logcat_monitor.sh

# 6. Просмотр логов
tail -f /opt/grani/logs/android/logcat_*.log
```

## Быстрый старт с Firebase Test Lab

```bash
# 1. Установка Firebase CLI
npm install -g firebase-tools

# 2. Вход
firebase login

# 3. Запуск теста
gcloud firebase test android run \
  --app /opt/grani/mobile-app/build/app/outputs/flutter-apk/app-debug.apk \
  --device model=Pixel4,version=30 \
  --timeout 5m \
  --results-bucket gs://your-bucket
```
