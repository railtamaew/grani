# Поведение экранов при появлении клавиатуры (mobile-app)

Экраны: **Ввод email** (`auth_email_screen.dart`), **Ввод PIN-кода** (`auth_code_screen.dart`).

## 1. Общий принцип

- Ничего не «прыгает» хаотично.
- Экран становится прокручиваемым; контент поднимается ровно настолько, чтобы активный инпут и CTA были видны.

## 2. Структура экрана (три зоны)

### Зона A — фиксированная верхняя

- **Содержимое:** логотип (и при наличии — меню / кнопка «Поделиться»).
- **Поведение:** не двигается, не скроллится, всегда на месте.
- **В коде:** первый дочерний виджет `Column` вне `SingleChildScrollView` (например, `Padding` + `Center(Logo)`).

### Зона B — основной контент (скроллируемый)

- **Содержимое:** заголовок, подзаголовок, поле ввода (email / PIN), блок ошибок, кнопка действия (Отправить код / подтверждение по PIN).
- **Поведение при клавиатуре:** контейнер скроллируется; активное поле при фокусе подтягивается в верхнюю треть экрана; кнопка действия всегда остаётся видимой (не фиксируется к низу экрана).
- **В коде:** единственный `SingleChildScrollView` внутри `Expanded`; `padding.bottom = MediaQuery.padding.bottom + MediaQuery.viewInsets.bottom`.

### Зона C — нижняя (вторичная)

- **Содержимое:** «Подробнее о конфиденциальности» (email), таймер / resend (PIN).
- **Поведение:** при клавиатуре уходит за скролл или скрывается; не перекрывает CTA.
- **В коде:** последние дочерние виджеты внутри того же `SingleChildScrollView` (после кнопки CTA).

## 3. Реализация во Flutter

- **Scaffold:** `resizeToAvoidBottomInset: true`.
- **Корень body:** градиент, затем `Column`:
  - Зона A: фиксированный блок с логотипом.
  - `Expanded(child: SingleChildScrollView(...))` с padding:
    - `EdgeInsets.fromLTRB(..., bottom: MediaQuery.of(context).padding.bottom + MediaQuery.of(context).viewInsets.bottom)`.
- **Фокус и скролл:** у поля ввода (email / PIN) — `FocusNode`; при фокусе вызывать `Scrollable.ensureVisible(context, alignment: 0.25)` (поле в верхней трети), чтобы и инпут, и CTA оставались в зоне видимости.

## 4. Запрещённые паттерны

- Кнопка под клавиатурой (фиксированная к низу экрана при открытой клавиатуре).
- Фиксированная кнопка при скрытом инпуте.
- Автоматический resize всего экрана без скролла.
- Сжатие контента до нечитаемого состояния.

## 5. UX-цель

Пользователь всегда видит, куда вводит, и что нажимать; экран не «ломается» при появлении клавиатуры.
