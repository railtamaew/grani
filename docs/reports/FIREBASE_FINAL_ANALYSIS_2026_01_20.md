## ❌ Корневая причина

**UI Automator не имеет доступа к системным диалогам в Firebase Test Lab.**

**Причины:**

1. **Процессная изоляция:**
   - VPN диалог запускается в процессе `com.android.vpndialogs` (системное приложение)
   - UI Automator работает в контексте тестового приложения
   - UI Automator может не видеть элементы из других процессов

2. **Ограничения Firebase Test Lab:**
   - Firebase Test Lab может ограничивать доступ к системным диалогам
   - Может быть политика безопасности, запрещающая автоматическое взаимодействие с системными диалогами

3. **Структура диалога:**
   - Кнопки могут быть не `android.widget.Button`
   - Могут быть кастомные View или другие компоненты
   - Могут требовать специальных разрешений для доступа

## 💡 Решения

### Решение 1: Использовать Accessibility Service (рекомендуется)

**Преимущества:**
- Может видеть элементы из всех процессов
- Более надежный доступ к системным диалогам
- Работает на уровне системы

**Реализация:**

1. **Создать Accessibility Service:**
   ```kotlin
   class VpnPermissionAccessibilityService : AccessibilityService() {
       override fun onAccessibilityEvent(event: AccessibilityEvent) {
           if (event.packageName == "com.android.vpndialogs") {
               // Найти и нажать кнопку в диалоге
           }
       }
   }
   ```

2. **Зарегистрировать в AndroidManifest.xml:**
   ```xml
   <service
       android:name=".VpnPermissionAccessibilityService"
       android:permission="android.permission.BIND_ACCESSIBILITY_SERVICE">
       <intent-filter>
           <action android:name="android.accessibilityservice.AccessibilityService" />
       </intent-filter>
   </service>
   ```

3. **Включить в настройках (требует ручного действия):**
   - Нужно включить Accessibility Service вручную
   - Или использовать ADB команду для включения

**Проблема:** Требует включения Accessibility Service, что может быть сложно в Firebase Test Lab.

---

### Решение 2: Использовать ADB команды для выдачи разрешения

**Попытка выдать разрешение через ADB:**

```bash
# Проверить текущие разрешения
adb shell dumpsys package com.granivpn.mobile | grep -i vpn

# Попытка выдать разрешение (может не работать)
adb shell pm grant com.granivpn.mobile android.permission.BIND_VPN_SERVICE
```

**Проблема:** VPN разрешение требует интерактивного подтверждения и не может быть выдано через `pm grant`.
