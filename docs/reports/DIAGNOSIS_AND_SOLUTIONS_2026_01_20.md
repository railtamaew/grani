# Диагностика проблем и решения - 20 января 2026

## 🔍 Выполненная диагностика

### Проблема 1: Эмулятор не запускается

**Ошибка:**
```
FATAL | Broken AVD system path. Check your ANDROID_SDK_ROOT value [/opt/android-sdk]!
```

**Причина:**
- Эмулятор не может найти SDK root, хотя путь правильный
- Это известная проблема Android Emulator версии 36.3.10.0

**Статус:** ❌ Не решено

**Влияние:**
- Блокирует запуск VNC сервера (порт 5901)
- Блокирует запуск noVNC (порт 6080)
- Блокирует всю систему тестирования

**Попытки решения:**
1. ✅ Проверена структура SDK - все компоненты на месте
2. ✅ Проверены переменные окружения - ANDROID_SDK_ROOT установлен
3. ✅ Создан AVD grani_optimized с правильными параметрами
4. ❌ Проблема сохраняется

### Проблема 2: noVNC (websockify) не установлен

**Ошибка:**
```
python3: can't open file '/opt/grani/emulator_web/utils/websockify/websockify.py': [Errno 2] No such file or directory
```

**Причина:**
- websockify не был установлен в директории emulator_web

**Статус:** ✅ Решено (websockify установлен)

**Влияние:**
- Блокировал запуск noVNC веб-интерфейса

### Проблема 3: URL granilink.com:6080 не доступен

**Диагностика:**
- ✅ DNS: granilink.com разрешается на 159.223.199.122
- ✅ Firewall: Порт 6080 открыт в UFW
- ❌ Порт 6080: Не слушается (websockify не запущен)
- ❌ Порт 5901: Не слушается (эмулятор не запущен)

**Причина:**
- websockify не запущен, потому что VNC сервер (эмулятор) не работает
- Цепочка зависимостей: Эмулятор → VNC → websockify → noVNC

**Статус:** ⚠️ Частично решено (websockify установлен, но не может запуститься без VNC)

**Вариант C: Попробовать другую версию эмулятора**

```bash

sdkmanager --update

# Или использовать более старую версию

```

**Вариант D: Исправить проблему с SDK path**

Проблема может быть в том, что эмулятор ищет SDK в другом месте. Попробовать:

```bash
# Создать символическую ссылку
ln -s /opt/android-sdk /root/android-sdk

# Или изменить ANDROID_SDK_ROOT
export ANDROID_SDK_ROOT=/root/android-sdk
```
