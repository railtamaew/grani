# Исправление автоматического создания токена при регистрации

## Проблема
Токен не создавался автоматически при регистрации из-за неправильного вызова метода `create_access_token`.

## Исправление

### Изменения в `backend/api/auth.py`:

**Было:**
```python
# Создаем токен
auth_service = AuthService(db)
token = auth_service.create_access_token(user.id)
```

**Стало:**
```python
# Создаем токен
token = AuthService.create_access_token(user.id)
```

### Объяснение
Метод `create_access_token` является статическим методом класса `AuthService` и не требует создания экземпляра класса. Использование статического метода напрямую более корректно и эффективно.

## Текущий процесс регистрации

1. **Пользователь запрашивает код** (`/auth/send-code`):
   - Если пользователя нет - создается новый пользователь
   - Генерируется код подтверждения
   - Код отправляется на email
   - **Токен НЕ создается на этом этапе** (это правильно)

2. **Пользователь вводит код** (`/auth/verify-code`):
   - Проверяется код
   - Если код верный:
     - Пользователь верифицируется (`is_verified = True`)
     - **Создается JWT токен** ✅
     - Токен возвращается в ответе:
       ```json
       {
         "ok": true,
         "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
         "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
         "user": {
           "id": 1,
           "email": "user@example.com",
           ...
         },
         ...
       }
       ```

3. **Мобильное приложение** (`mobile-app/lib/services/auth_service.dart`):
   - Получает токен из ответа (строка 840)
   - Сохраняет токен через `_saveToken()` (строка 868-875)
   - Токен сохраняется в SecureStorage и SharedPreferences

## Проверка

✅ Токен создается корректно при верификации кода
✅ Токен возвращается в ответе API
✅ Мобильное приложение сохраняет токен автоматически

## Результат

После исправления:
- ✅ Токен автоматически создается при успешной верификации кода
- ✅ Токен автоматически возвращается в ответе `/auth/verify-code`
- ✅ Мобильное приложение автоматически сохраняет токен
- ✅ Пользователь может сразу использовать приложение после регистрации

## Важно

Токен создается **только после успешной верификации кода**, а не при запросе кода. Это правильное поведение для безопасности.
