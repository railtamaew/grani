## 🔍 Диагностика падающих тестов

### Проблема 1: Тесты пропускаются из-за разрешений VPN

**Симптомы:**
- Тесты используют `assumeTrue("VPN permission not granted", ensureVpnPermission(context))`
- При запуске через Gradle тесты пропускаются (SKIPPED)
- При ручном запуске через `am instrument` тесты проходят

**Причина:**
- `ensureVpnPermission()` не может автоматически принять разрешение при запуске через Gradle
- UI Automator не работает в контексте Gradle тестов

**Решение:**
1. Использовать ручной режим (через браузер noVNC)
2. Улучшить `ensureVpnPermission()` - добавить больше методов
3. Предварительно выдавать разрешение перед запуском тестов

### Проблема 2: WireGuard тест падает с TimeoutException

**Симптомы:**
- `GoBackend.setStateInternal` → `TimeoutException`
- VPN не стартует на эмуляторе

**Возможные причины:**
- Проблема с WireGuard библиотекой на эмуляторе
- Проблема с конфигурацией
- Проблема с разрешениями

**Решение:**
- Собрать логи при запуске WireGuard
- Проверить конфигурацию
- Попробовать на физическом устройстве

### Проблема 3: Недостаточно диагностики

**Симптомы:**
- Нет подробных логов при выполнении тестов
- Непонятно на каком этапе происходит ошибка

**Решение:**
- Добавить подробное логирование в тесты
- Собрать логи при выполнении
- Проанализировать ошибки

### Цель: Запустить приложение на эмуляторе и получить логи

### Шаг 1: Запуск системы тестирования
```bash
/opt/grani/scripts/start_combined_testing.sh manual
```

### Шаг 2: Установка и запуск приложения
```bash
cd /opt/grani/mobile-app
flutter build apk --debug
adb install build/app/outputs/flutter-apk/app-debug.apk
adb shell am start -n com.granivpn.mobile/.MainActivity
```

### Шаг 3: Мониторинг логов
```bash
tail -f /opt/grani/logs/android/logcat_*.log
```

### Шаг 4: Тестирование подключения
1. В браузере (noVNC) открыть приложение
2. Выбрать сервер и протокол
3. Нажать "Подключиться"
4. Дать разрешение VPN через браузер
