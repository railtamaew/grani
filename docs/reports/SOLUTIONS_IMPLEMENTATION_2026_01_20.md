## Выполненные задачи

### ✅ 1. Скрипт диагностики и исправления эмулятора

**Файл:** `/opt/grani/scripts/fix_emulator.sh`

**Функциональность:**
- Проверка GLIBC версии
- Проверка наличия библиотек (X11, Qt, и др.)
- Проверка LD_LIBRARY_PATH
- Проверка системных ресурсов (RAM, CPU, диск)
- Проверка KVM поддержки
- Проверка конфигурации AVD
- Автоматическое исправление проблем
- Тестовый запуск эмулятора

**Использование:**
```bash
# Только диагностика
/opt/grani/scripts/fix_emulator.sh --diagnose-only

# Диагностика и исправление
/opt/grani/scripts/fix_emulator.sh
```

**Исправления:**
- Автоматическая установка недостающих библиотек
- Обновление LD_LIBRARY_PATH с правильными путями
- Проверка и исправление конфигурации AVD

### ✅ 2. Система удаленного доступа к физическому устройству

#### 2.1. Скрипт для клиентской машины

**Файл:** `/opt/grani/scripts/setup_adb_remote_client.sh`

**Функциональность:**
- Автоматический поиск подключенного устройства
- Получение IP адреса устройства
- Включение TCP/IP режима ADB
- Подключение к устройству по сети
- Создание SSH туннеля к серверу

**Использование:**
```bash
# На вашем компьютере:
./setup_adb_remote_client.sh

# С указанием IP устройства:
./setup_adb_remote_client.sh --device-ip 192.168.1.100

# С указанием сервера:
./setup_adb_remote_client.sh --server granilink.com --server-user root
```

#### 2.2. Скрипт для сервера

**Файл:** `/opt/grani/scripts/setup_adb_remote_server.sh`

**Функциональность:**
- Проверка подключения через SSH туннель
- Автоматическое подключение к устройству
- Проверка доступных устройств
- Вывод инструкций по использованию

**Использование:**
```bash
# На сервере:
/opt/grani/scripts/setup_adb_remote_server.sh

# С автоматическим подключением:
/opt/grani/scripts/setup_adb_remote_server.sh 5555 true
```

#### 2.3. Скрипт подключения к удаленному устройству

**Файл:** `/opt/grani/scripts/connect_remote_device.sh`

**Функциональность:**
- Подключение через SSH туннель (localhost:5555)
- Прямое подключение по IP:PORT
- Сканирование сети для поиска устройств
- Проверка подключения и вывод информации об устройстве

**Использование:**
```bash
# Через SSH туннель:
/opt/grani/scripts/connect_remote_device.sh

# Прямое подключение:
/opt/grani/scripts/connect_remote_device.sh 192.168.1.100:5555

# Сканирование сети:
/opt/grani/scripts/connect_remote_device.sh --scan
```

### ✅ 3. Автоматический сбор логов с удаленного устройства

**Файл:** `/opt/grani/scripts/collect_remote_logs.sh`

**Функциональность:**
- Автоматический поиск удаленного устройства
- Проверка подключения
- Очистка старых логов
- Сбор логов с фильтрацией по пакету и ключевым словам
- Сохранение логов в файл
- Автоматическая остановка после указанного времени

**Использование:**
```bash
# Сбор логов в течение 5 минут (по умолчанию):
/opt/grani/scripts/collect_remote_logs.sh

# Сбор логов в течение 10 минут:
/opt/grani/scripts/collect_remote_logs.sh 600

# С указанием устройства:
/opt/grani/scripts/collect_remote_logs.sh 300 localhost:5555
```

**Логи сохраняются в:**
- `/opt/grani/logs/android/remote/remote_logcat_*.log`

### ✅ 4. Обновление скрипта запуска эмулятора

**Файл:** `/opt/grani/scripts/start_emulator_with_vnc.sh`

**Изменения:**
- Добавлен экспорт LD_LIBRARY_PATH с правильными путями
- Исправление применяется автоматически при запуске

**Пути в LD_LIBRARY_PATH:**
- `/usr/lib/x86_64-linux-gnu` - системные библиотеки
- `/opt/android-sdk/emulator/lib64` - библиотеки эмулятора
- `/opt/android-sdk/emulator/lib64/qt/lib` - Qt библиотеки


### Быстрый старт для удаленного устройства

**На вашем компьютере:**
```bash
# 1. Скачайте скрипт
scp root@granilink.com:/opt/grani/scripts/setup_adb_remote_client.sh ./

# 2. Запустите
chmod +x setup_adb_remote_client.sh
./setup_adb_remote_client.sh
```

**На сервере:**
```bash
# 1. Подключитесь к устройству
/opt/grani/scripts/connect_remote_device.sh

# 2. Запустите приложение с логированием
/opt/grani/scripts/run_app_with_logging.sh

# 3. Или соберите логи автоматически
/opt/grani/scripts/collect_remote_logs.sh 300
```

**Решение:**
