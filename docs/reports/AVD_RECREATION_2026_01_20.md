# Пересоздание AVD с оптимальными параметрами - 20 января 2026

## ✅ Выполнено

Создан скрипт для пересоздания AVD с оптимальными параметрами для стабильной работы на сервере.

## 📋 Созданный скрипт

**Файл:** `/opt/grani/scripts/recreate_avd_optimized.sh`

**Функции:**
1. Проверка ресурсов сервера
2. Установка system image (если нужно)
3. Удаление старого AVD (если существует)
4. Остановка запущенных эмуляторов
5. Создание нового AVD
6. Настройка config.ini с оптимальными параметрами
7. Проверка конфигурации

## 🎯 Оптимальные параметры AVD

### Ресурсы
- **RAM:** 1536MB (минимум для стабильности)
- **CPU:** 1 core (достаточно для работы без зависаний)
- **Disk:** 2048MB (минимальный размер data partition)

### Отключенные функции
- **Audio:** отключен (уменьшает нагрузку)
- **Camera:** отключена (не нужна для тестирования VPN)
- **Сенсоры:** отключены (акселерометр, гироскоп, GPS и т.д.)
- **Play Store:** отключен (не нужен)

### GPU
- **Режим:** `swiftshader_indirect` (работает без KVM, стабильнее)

### Загрузка
- **Snapshots:** отключены (избегаем проблем с поврежденными снимками)
- **Fast boot:** включен (быстрая загрузка)

## 📊 Сравнение параметров

| Параметр | Старый AVD | Новый AVD (grani_optimized) |
|----------|------------|------------------------------|
| RAM | 2048MB | 1536MB |
| CPU cores | 2 | 1 |
| Data partition | 3072MB | 2048MB |
| Audio | включен | отключен |
| Camera | включена | отключена |
| GPU mode | auto | swiftshader_indirect |
| Snapshots | включены | отключены |

## 🚀 Использование

### Создание нового AVD
```bash
/opt/grani/scripts/recreate_avd_optimized.sh [AVD_NAME]
```

По умолчанию создается `grani_optimized`.

### Запуск с новым AVD
```bash
export AVD_NAME=grani_optimized
/opt/grani/scripts/start_combined_testing.sh manual
```

### Прямой запуск эмулятора
```bash
export AVD_NAME=grani_optimized
/opt/grani/scripts/start_emulator_with_vnc.sh
```

## 🔧 Обновления скриптов

### `start_emulator_with_vnc.sh`
- Добавлен параметр `-wipe-data` для чистой загрузки
- Параметры соответствуют настройкам AVD в config.ini

## 📝 Практики, использованные при создании

1. **Минимальные ресурсы** - используем только необходимое для стабильной работы
2. **Отключение ненужных функций** - уменьшаем нагрузку на систему
3. **Стабильный GPU режим** - swiftshader_indirect работает без KVM
4. **Отключение snapshots** - избегаем проблем с поврежденными снимками
5. **Ручная настройка config.ini** - полный контроль над параметрами

## ⚠️ Известные ограничения

1. **Без KVM** - эмулятор работает медленнее, но стабильнее
2. **Минимальные ресурсы** - может быть медленнее при сложных операциях
3. **Отключенные функции** - некоторые функции Android недоступны
