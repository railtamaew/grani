## Возможные причины проблемы

### 1. Проблема с DNS на клиенте

**Проверка:**
```powershell
# В PowerShell на Windows
nslookup google.com
```

**Решение:**
- Убедитесь, что DNS серверы в wg0.conf правильные: `8.8.8.8, 8.8.4.4`
- Попробуйте открыть сайт по IP: `http://8.8.8.8` (должен показать страницу Google)

### 2. Файрвол Windows блокирует трафик

**Проверка:**
- Отключите файрвол Windows временно
- Или добавьте исключение для WireGuard

**Решение:**
```powershell
# Добавить исключение для WireGuard
New-NetFirewallRule -DisplayName "WireGuard" -Direction Outbound -Program "C:\Program Files\WireGuard\wireguard.exe" -Action Allow
```

### 3. Трафик не маршрутизируется через VPN

**Проверка:**
```powershell
# Проверьте маршруты
route print

# Должен быть маршрут:
# 0.0.0.0 0.0.0.0 10.0.0.1 ... wg0
```

**Решение:**
- Убедитесь, что `AllowedIPs = 0.0.0.0/0` в wg0.conf
- Переподключитесь к VPN

### 4. Проблема с MTU

**Проверка:**
```powershell
# Попробуйте ping с меньшим размером пакета
ping -l 1200 8.8.8.8
```

**Решение:**
- Если ping работает с меньшим размером, уменьшите MTU в wg0.conf до 1280

## Пошаговая диагностика

### Шаг 1: Проверка подключения

1. Откройте WireGuard
2. Убедитесь, что туннель `wg0` подключен (зеленая галочка)
3. Проверьте "Последнее рукопожатие" - должно быть недавно

### Шаг 2: Проверка ping

```powershell
# В PowerShell
ping 8.8.8.8
ping 10.0.0.1  # IP сервера в VPN сети
```

**Ожидаемый результат:**
- `8.8.8.8` должен отвечать (если VPN работает)
- `10.0.0.1` должен отвечать (это сервер в VPN сети)

### Шаг 3: Проверка DNS

```powershell
# Проверка DNS
nslookup google.com 8.8.8.8
```

**Ожидаемый результат:**
- Должен вернуть IP адрес google.com

### Шаг 4: Проверка трафика

В WireGuard проверьте:
- "Передача" - должны увеличиваться и отправленные, и полученные байты
- Если полученные байты не увеличиваются - проблема на сервере
- Если отправленные не увеличиваются - проблема на клиенте

### Шаг 5: Проверка маршрутов

```powershell
# Проверьте маршруты
route print | findstr "0.0.0.0"
```

**Ожидаемый результат:**
- Должен быть маршрут через `10.0.0.1` (VPN сервер)

## Быстрое решение

1. **Переподключитесь к VPN:**
   - Отключите туннель в WireGuard
   - Подключите снова

2. **Проверьте DNS:**
   - Попробуйте открыть `http://8.8.8.8` в браузере
   - Если открывается - проблема с DNS

3. **Проверьте файрвол:**
   - Временно отключите файрвол Windows
   - Попробуйте открыть сайт

4. **Проверьте маршруты:**
   - Убедитесь, что весь трафик идет через VPN
   - Проверьте `route print`

## Дополнительная диагностика на сервере

Если проблема сохраняется, можно проверить на сервере:

```bash
# Подключитесь к серверу
ssh root@45.12.132.94

# Проверьте статус WireGuard
wg show

# Проверьте счетчики пакетов
iptables -L FORWARD -n -v | grep wg0
iptables -t nat -L POSTROUTING -n -v | grep 10.0.0.0

# Проверьте логи
journalctl -u wg-quick@wg0 -f
```

3. Убедитесь, что файрвол Windows не блокирует трафик
4. Попробуйте другой браузер или приложение
