# Исправление конфигурации XRay для libXray

## Дата: 2026-01-03

## Проблема
libXray не мог прочитать конфигурацию XRay, выдавая ошибку:
```
failed to read config file > EOF
```

## Причина
Конфигурация содержала `dokodemo-door` inbound с `port: 0`, который libXray не поддерживает для работы с VPN интерфейсом Android.

## Решение
1. **Убран `dokodemo-door` inbound** - libXray не поддерживает этот протокол с `port: 0` для VPN интерфейса
2. **Используется только SOCKS inbound** - порт 10808, протокол SOCKS5 с поддержкой UDP
3. **Добавлена валидация JSON** - проверка валидности конфигурации перед передачей в libXray

## Измененные файлы
- `mobile-app/android/app/src/main/kotlin/com/granivpn/mobile/XrayConfig.kt`
  - Убран `dokodemo-door` inbound
  - Оставлен только SOCKS inbound (порт 10808)
  - Добавлена валидация JSON конфигурации

## Следующие шаги

Трафик из VPN интерфейса должен перенаправляться через локальный SOCKS прокси XRay (порт 10808). Это требует дополнительной реализации в `VpnService.kt`.
