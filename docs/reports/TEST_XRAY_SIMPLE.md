# Простая инструкция: Как протестировать XRay

## Самый простой способ

### 1. Запустите тестовый скрипт:

```bash
cd /opt/grani/backend
python3 scripts/vpn_tester.py
```

### 2. В меню выполните следующие действия:

```
Выберите действие: 5  # Отключиться от VPN (если подключены)
Выберите действие: 3  # Подключиться к VPN
Введите ID сервера: 1
Введите протокол [wireguard]: xray_vless
```

### 3. Результат:

Вы получите конфигурацию XRay, которую можно использовать в клиенте.

## Альтернатива: Через меню скрипта

1. **Запустите:** `python3 scripts/vpn_tester.py`

2. **Выберите пункты по порядку:**
   - `1` - Получить токен
   - `2` - Получить список серверов (посмотрите доступные протоколы)
   - `5` - Отключиться (если уже подключены)
   - `3` - Подключиться к VPN
     - Введите: `1` (ID сервера)
     - Введите: `xray_vless` (протокол)

3. **Получите конфигурацию** и используйте её в клиенте XRay

## Что делать с конфигурацией?

### Для тестирования на компьютере:

1. **Скачайте клиент XRay:**
   - Windows: [v2rayN](https://github.com/2dust/v2rayN/releases)
   - Linux/Mac: [Xray-core](https://github.com/XTLS/Xray-core/releases)

2. **Импортируйте конфигурацию:**
   - В v2rayN: Импорт → Из файла → выберите `xray_config.json`
   - Или скопируйте URI (vless://...) и импортируйте

3. **Подключитесь** и проверьте интернет

### Для мобильного приложения:

Конфигурация будет использована автоматически при выборе протокола `xray_vless` или `xray_vmess`.

## Быстрая команда (одной строкой)

```bash
cd /opt/grani/backend && echo -e "1\n3\n1\nxray_vless" | python3 scripts/vpn_tester.py
```

(Но лучше использовать интерактивный режим для просмотра результатов)

## Проверка работы

После подключения:
1. Откройте браузер
2. Попробуйте открыть любой сайт
3. Проверьте IP: `curl ifconfig.me` (должен показать IP сервера 45.12.132.94)

## Если что-то не работает

1. Проверьте статус XRay на сервере:
   ```bash
   ssh root@45.12.132.94 "systemctl status xray"
   ```

2. Проверьте порт:
   ```bash
   ssh root@45.12.132.94 "ss -tlnp | grep 4443"
   ```

3. Проверьте логи:
   ```bash
   ssh root@45.12.132.94 "journalctl -u xray -n 20"
   ```
