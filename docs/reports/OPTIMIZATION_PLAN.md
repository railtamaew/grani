# План дополнительной оптимизации проекта

## Текущее состояние

### Размеры директорий:
- `mobile-app/`: 6.0GB (основной размер)
- `admin-panel/`: 982MB
- `backend/`: 1.9MB
- `emulator_web/`: 2.9MB
- `logs/`: 240KB
- `docs/`: 300KB

### Проблемные области:

1. **Build артефакты (1.3GB+)**:
   - `mobile-app/build/`: 1.3GB
   - `mobile-app/.dart_tool/`: 51MB
   - `admin-panel/build/`: 1.1MB

2. **Дубликаты APK (116MB)**:
   - `granivpn-release.apk`: 58MB
   - `mobile-app/build/app/outputs/apk/release/app-release.apk`: 58MB
   - `mobile-app/build/app/outputs/flutter-apk/app-release.apk`: 58MB

3. **Python кэш (20KB)**:
   - `backend/__pycache__/`: 20KB

4. **Старые логи (240KB)**:
   - Логи старше 7 дней: 18 файлов

5. **Node modules (980MB)**:
   - `admin-panel/node_modules/`: 980MB (необходимы для работы)

## Рекомендации по оптимизации

### 1. Очистка build артефактов (экономия ~1.35GB)
**Действие:** Удалить директории сборки
- `mobile-app/build/` - можно пересобрать при необходимости
- `mobile-app/.dart_tool/` - кэш Dart, пересоздается автоматически
- `admin-panel/build/` - можно пересобрать

**Риск:** Низкий - файлы можно пересобрать

### 2. Удаление дубликатов APK (экономия ~116MB)
**Действие:** Оставить только последний APK
- Удалить старые APK из `mobile-app/build/`
- Оставить только `granivpn-release.apk` в корне

**Риск:** Низкий - можно пересобрать

### 3. Очистка Python кэша (экономия ~20KB)
**Действие:** Удалить `__pycache__` директории
- `backend/__pycache__/` и все поддиректории

**Риск:** Низкий - пересоздается автоматически

### 4. Очистка старых логов (экономия ~240KB)
**Действие:** Удалить логи старше 7 дней
- `logs/*.log` старше 7 дней
- Оставить только последние логи

**Риск:** Низкий - логи для отладки

### 5. Проверка неиспользуемых зависимостей
**Действие:** Анализ зависимостей
- Проверить неиспользуемые пакеты в `package.json`
- Проверить неиспользуемые пакеты в `pubspec.yaml`
- Удалить неиспользуемые зависимости

**Риск:** Средний - нужно проверить использование

### 6. Оптимизация структуры проекта
**Действие:** Проверка дублирования кода
- Найти дублирующиеся функции
- Объединить общий код

**Риск:** Низкий - улучшение качества кода
