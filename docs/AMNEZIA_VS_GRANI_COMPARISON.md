# Сравнительный анализ: AmneziaVPN vs GRANI VPN

## Дата анализа: 2025-01-14

---

## 1. Архитектура сервера

### AmneziaVPN
- **Подход**: Docker-based архитектура
  - Каждый протокол работает в отдельном Docker контейнере
  - Автоматическая установка через SSH из клиентского приложения
  - Требования к серверу: Ubuntu 22.04/24.04, Debian 12/13, x86_64, KVM, IPv4, systemd
  - Минимум 1GB RAM (рекомендуется 2GB+)

- **Управление**: 
  - Клиентское приложение управляет сервером через SSH
  - Автоматическая генерация ключей и сертификатов
  - Установка Docker и контейнеров протоколов

### GRANI VPN
- **Подход**: Гибридная архитектура (локальная + удаленная)
  - Прямая установка протоколов на сервер (WireGuard, Xray, OpenVPN)
  - Управление через SSH для удаленных серверов
  - Python backend с менеджерами протоколов
  - Поддержка как локальных, так и удаленных серверов

- **Управление**:
  - Backend API управляет серверами через SSH
  - `RemoteVPNManager` для удаленных серверов
  - `WireGuardManager`, `XrayManager`, `OpenVPNManager` для локальных
  - Автоматическая генерация конфигураций

**Ключевые отличия:**
- ✅ AmneziaVPN: Изоляция через Docker, проще масштабирование
- ✅ GRANI VPN: Прямая установка, меньше overhead, больше контроля

## 2. Поддерживаемые протоколы

### AmneziaVPN

| Протокол | Обфускация | Описание |
|----------|-----------|----------|
| **OpenVPN** | Cloak plugin | TCP/UDP, TLS-based, с маскировкой трафика |
| **WireGuard** | Нет (стандартный) | UDP, легко обнаруживается DPI |
| **AmneziaWG** | Да (кастомный) | Обфусцированный WireGuard с рандомизацией |
| **IKEv2 (IPsec)** | Нет | UDP 500/4500, уязвим к блокировкам |
| **Shadowsocks** | Встроенная | TCP, SOCKS5, похож на HTTPS |
| **XRay Reality** | Встроенная | TLS 443, маскировка под реальный сайт |

**AmneziaWG особенности:**
- Рандомизация заголовков пакетов
- Переменная длина handshake (S1, S2 до 64 байт)
- Signature packets (I1-I5) перед handshake
- Junk-train пакеты для маскировки
- Обфускация Under-Load пакетов
- Сохранение криптографии WireGuard (Curve25519, ChaCha20-Poly1305)

### GRANI VPN

| Протокол | Обфускация | Статус |
|----------|-----------|--------|
| **WireGuard** | Нет | ✅ Полностью реализован |
| **GRANIWG** | Планируется (udp2raw) | ⚠️ Частично (заглушка) |
| **XRay VLESS** | Нет | ✅ Реализован |
| **XRay VMESS** | Нет | ✅ Реализован |
| **XRay Reality** | Встроенная | ✅ Реализован |
| **OpenVPN over Cloak** | Cloak | ✅ Реализован |

**Текущее состояние GRANIWG:**
```python
# backend/services/wireguard_manager.py
def create_graniwg_client_config(...):
    # Пока только заглушка с комментариями
    # Реальная обфускация не реализована
```

**Ключевые отличия:**
- ✅ AmneziaVPN: Больше протоколов (IKEv2, Shadowsocks)
- ✅ AmneziaVPN: Более продвинутая обфускация (AmneziaWG)
- ✅ GRANI VPN: XRay протоколы полностью реализованы
- ⚠️ GRANI VPN: GRANIWG требует доработки

## 3. Обфускация и антидетекция

### AmneziaVPN

**AmneziaWG (обфусцированный WireGuard):**
- Рандомизация всех типов пакетов (Init, Response, Data, Under-Load)
- Переменная длина handshake (добавление S1, S2 до 64 байт)
- Signature packets (I1-I5) с Custom Protocol Signatures
- Junk-train: цепочка рандомизированных пакетов (Jc штук, размер Jmin-Jmax)
- Обфускация keep-alive пакетов
- Импersonation протоколов (QUIC, DNS)

**Cloak:**
- Маскировка OpenVPN/Shadowsocks под обычный веб-трафик
- Изменение метаданных пакетов
- Защита от активного зондирования

**XRay Reality:**
- TLS handshake detection
- Редирект не-клиентских handshake на реальный сайт
- Легитимные сертификаты

### GRANI VPN

**GRANIWG:**
- ⚠️ Планируется: udp2raw обфускация
- ⚠️ Текущее состояние: только заглушка в коде
- Не реализована рандомизация пакетов
- Не реализованы signature/junk packets

**XRay Reality:**
- ✅ Полностью реализован
- ✅ Поддержка SNI, dest, short ID
- ✅ Генерация публичного/приватного ключей

**OpenVPN over Cloak:**
- ✅ Реализован
- ✅ Интеграция с Cloak plugin
- ✅ Конфигурация UID, ключей, mask site

**Ключевые отличия:**
- ❌ GRANI VPN: GRANIWG требует полной реализации обфускации
- ✅ GRANI VPN: XRay Reality и Cloak работают
- ✅ AmneziaVPN: Более продвинутая обфускация WireGuard

## 4. Управление серверами

### AmneziaVPN

**Клиент → Сервер:**
- Клиентское приложение подключается к серверу через SSH
- Автоматическая установка Docker
- Запуск контейнеров протоколов
- Генерация ключей и сертификатов
- Настройка конфигураций

**Особенности:**
- Self-hosted модель (пользователь предоставляет VPS)
- Полная автоматизация через клиент
- Изоляция протоколов в контейнерах

### GRANI VPN

**Backend → Сервер:**
- Python backend управляет серверами
- `RemoteVPNManager` для SSH операций
- `SSHManager` для выполнения команд
- Поддержка SSH ключей и паролей
- Управление конфигурациями через файловую систему

**Особенности:**
- Централизованное управление через backend
- Поддержка множества серверов
- Гибридная модель (локальные + удаленные)
- Мониторинг и статистика

**Код управления:**
```python
# backend/services/remote_vpn_manager.py
class RemoteVPNManager:
    def add_wireguard_peer(self, server, device, client_ip):
        # SSH подключение
        # Чтение/запись конфигурации
        # Перезагрузка WireGuard
```

**Ключевые отличия:**
- ✅ AmneziaVPN: Проще для пользователя (всё через клиент)
- ✅ GRANI VPN: Централизованное управление, лучше для провайдера
- ✅ GRANI VPN: Мониторинг и статистика встроены

## 5. Конфигурация клиентов

### AmneziaVPN

**Формат:**
- JSON конфигурации для каждого протокола
- Экспорт конфигураций для разных платформ
- Возможность поделиться конфигурацией (guest access)

**Особенности:**
- Абстрактный базовый класс `VpnProtocol`
- Конкретные реализации для каждого протокола
- UI для управления конфигурациями

### GRANI VPN

**Формат:**
- WireGuard: стандартный `.conf` формат
- XRay: URL формат (`vless://`, `vmess://`)
- OpenVPN: стандартный `.conf` с Cloak plugin

**Генерация конфигураций:**
```python
# WireGuard
config = f"""[Interface]
PrivateKey = {device.wireguard_private_key}
Address = {client_ip}/32
DNS = 8.8.8.8, 8.8.4.4
[Peer]
PublicKey = {server.wireguard_public_key}
Endpoint = {server.ip_address}:{server.wireguard_port}
AllowedIPs = 0.0.0.0/0
"""

# XRay VLESS
vless://{uuid}@{address}:{port}?security=none&type=tcp#GRANI
```

**Ключевые отличия:**
- ✅ AmneziaVPN: JSON формат, более структурированный
- ✅ GRANI VPN: Стандартные форматы, совместимость с другими клиентами

## 6. Безопасность и криптография

### AmneziaVPN

**WireGuard/AmneziaWG:**
- Curve25519 для key exchange
- ChaCha20-Poly1305 для шифрования
- Криптография идентична стандартному WireGuard
- Обфускация только на транспортном уровне

**OpenVPN:**
- TLS-based
- Поддержка различных cipher suites
- Cloak для дополнительной маскировки

**XRay Reality:**
- TLS handshake
- Легитимные сертификаты
- Защита от активного зондирования

### GRANI VPN

**WireGuard:**
- ✅ Стандартная криптография WireGuard
- ✅ Curve25519, ChaCha20-Poly1305

**XRay:**
- ✅ VLESS/VMESS с различными методами шифрования
- ✅ REALITY с TLS маскировкой

**OpenVPN:**
- ✅ AES-256-GCM
- ✅ SHA256 для аутентификации
- ✅ Cloak plugin

**Ключевые отличия:**
- ✅ Оба используют проверенные криптографические алгоритмы
- ✅ Безопасность на высоком уровне в обоих решениях

## 7. Производительность

### AmneziaVPN

**Overhead:**
- Docker контейнеры добавляют небольшой overhead
- Обфускация увеличивает latency и bandwidth
- Junk packets и рандомизация увеличивают размер пакетов

**Оптимизация:**
- Минимальные контейнеры
- Эффективная обфускация (только при необходимости)

### GRANI VPN

**Overhead:**
- Прямая установка = меньше overhead
- Обфускация пока не реализована (GRANIWG)
- XRay и WireGuard работают нативно

**Оптимизация:**
- Нативная производительность
- Нет Docker overhead
- Эффективное использование ресурсов

**Ключевые отличия:**
- ✅ GRANI VPN: Меньше overhead (нет Docker)
- ⚠️ AmneziaVPN: Больше overhead, но лучше изоляция
