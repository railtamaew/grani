# Статус интеграции Xray протоколов

## Дата проверки: 2026-01-27

### ⚠️ Интеграция с мобильным приложением

**Статус:** Android — включено, требуется финальное тестирование; iOS — не поддерживается

#### Endpoints:

**Backend API:**
- Роутер: `/api/vpn/xray` (определен в `backend/api/xray.py`)
- Endpoint создания клиента: `POST /api/vpn/xray/create-client`
- Endpoint статистики: `GET /api/vpn/xray/stats?server_id=1`
- Endpoint статистики клиента: `GET /api/vpn/xray/client-stats/{client_id}`

**Мобильное приложение:**
- Базовый URL: `http://159.223.199.122:8000/api` (из `app_config.dart`)
- Используемый путь: `/vpn/xray/create-client` (из `vpn_service.dart`)
- **Полный URL:** `http://159.223.199.122:8000/api/vpn/xray/create-client` ✅

#### Проверка совместимости:

1. ⚠️ **Мобильное приложение** - Xray доступен на Android, скрыт на iOS
2. ✅ **Endpoint совпадает** - приложение использует правильный путь
3. ✅ **Метод запроса** - POST корректный
4. ✅ **Параметры запроса** - совпадают:
   - `server_id` (int)
   - `device_id` (string)
   - `protocol` (string: "xray_vless", "xray_vmess", "xray_reality")
5. ✅ **Формат ответа** - приложение ожидает:
   - `success` (bool)
   - `client_id` (string)
   - `config` (string или Map)
   - `json_config` (Map) - приоритетно используется
   - `ip_address` (string)
6. ✅ **Авторизация** - используется Bearer token

### ⚠️ Потенциальные проблемы:

1. **Базовый URL** - указан как `http://159.223.199.122:8000/api`
   - Нужно убедиться, что это правильный адрес сервера
