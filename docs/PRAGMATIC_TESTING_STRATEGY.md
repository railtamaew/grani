# Прагматичная стратегия тестирования VPN протоколов

## ❌ Проблема текущего подхода

1. **Flutter тестовое приложение: 150MB, сборка 30 минут**
2. **Каждое исправление = пересборка APK**
3. **Логи не работают**
4. **Мы вернулись к неэффективному процессу**

## ✅ Правильное решение

### Принцип: Тестировать логику БЕЗ сборки APK

---

## Этап 1: Unit-тесты для логики (СЕЙЧАС)

### Что тестировать:

1. **Парсинг конфигураций**
   - WireGuard конфигурации
   - Sing-box конфигурации
   - Валидация форматов

2. **Логика обработки**
   - Определение типа протокола
   - Преобразование конфигураций
   - Обработка ошибок

3. **Моки для системных вызовов**
   - TUN интерфейс (мок)
   - VPN разрешения (мок)
   - Системные сервисы (мок)

### Преимущества:
- ✅ Запуск: 10 секунд
- ✅ Не нужен APK
- ✅ Мгновенная обратная связь
- ✅ Можно запускать на сервере

### Где писать:
- `mobile-app/android/app/src/test/kotlin/` - для Kotlin кода
- `backend/tests/` - для backend логики

---

## Этап 2: Тестирование в основном приложении

### Стратегия:

1. **Исправляем код в основном приложении**
2. **Собираем только измененные модули** (не весь APK)
3. **Устанавливаем через ADB** (быстро)
4. **Тестируем**

### Быстрая установка:

```bash
# Собираем только измененный модуль
cd /opt/grani/mobile-app/android
./gradlew :app:assembleDebug

# Устанавливаем только измененный APK
adb install -r app/build/outputs/apk/debug/app-debug.apk

# Или используем hot reload (если возможно)
```

### Преимущества:
- ✅ Используем существующее приложение
- ✅ Не создаем новое
- ✅ Тестируем в реальных условиях
- ✅ Быстрее чем создавать новое приложение

---

## Этап 3: Интеграционные тесты (только при необходимости)

### Когда нужны:
- Только для проверки реального TUN интерфейса
- Только после успешных unit-тестов

### Что делать:
- Использовать основное приложение
- Добавить режим отладки
- Тестировать через ADB

---

## 📊 Сравнение подходов

| Подход | Время | Размер | Эффективность |
|--------|-------|--------|---------------|
| Flutter тестовое приложение | 30 мин | 150MB | ❌ Плохо |
| Unit-тесты | 10 сек | 0MB | ✅ Отлично |
| Основное приложение | 5-10 мин | 150MB | ✅ Хорошо |
| Нативное тестовое | 2-3 мин | 5MB | ⚠️ Можно, но не нужно |

## 🎯 План действий

### СЕЙЧАС:

1. **Удалить тестовое Flutter приложение** (неэффективно)
2. **Создать unit-тесты для:**
   - Парсинг WireGuard конфигураций
   - Парсинг Sing-box конфигураций
   - Логика определения протокола
   - Обработка ошибок

3. **Тестировать в основном приложении:**
   - Исправляем код
   - Собираем и устанавливаем
   - Тестируем

### НЕ ДЕЛАТЬ:

- ❌ Создавать новое приложение
- ❌ Собирать APK для каждого исправления
- ❌ Использовать Flutter для тестирования
