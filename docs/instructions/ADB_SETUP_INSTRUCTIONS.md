# Инструкция по настройке ADB over Network

## Ситуация

Устройство OnePlus подключено к вашему локальному компьютеру (Windows), а сервер находится удаленно. Для работы с сервера нужно настроить ADB over network.

## Шаг 1: Настройка на локальном компьютере (Windows)

### Вариант A: PowerShell скрипт (рекомендуется)

1. **Скачайте скрипт с сервера:**
   ```powershell
   # В PowerShell на вашем Windows компьютере
   scp user@server:/opt/grani/setup_adb_local_windows.ps1 ./
   ```

2. **Запустите скрипт:**
   ```powershell
   .\setup_adb_local_windows.ps1
   ```

   Скрипт автоматически:
   - Найдет подключенное устройство
   - Получит IP адрес
   - Включит TCP/IP режим
   - Подключится по сети

### Вариант B: Ручная настройка

1. **Убедитесь, что ADB установлен на Windows:**
   - Скачайте [Android SDK Platform Tools](https://developer.android.com/studio/releases/platform-tools)
   - Добавьте в PATH или используйте полный путь

2. **Проверьте подключение:**
   ```powershell
   adb devices
   ```
   Должно показать ваше устройство

3. **Узнайте IP адрес устройства:**
   - На устройстве: Настройки → Wi-Fi → текущая сеть → IP адрес
   - Или через ADB: `adb shell "getprop dhcp.wlan0.ipaddress"`

4. **Включите TCP/IP режим:**
   ```powershell
   adb tcpip 5555
   ```

5. **Подключитесь по сети:**
   ```powershell
   adb connect ВАШ_IP_АДРЕС:5555
   ```
   Например: `adb connect 192.168.1.100:5555`

6. **Проверьте:**
   ```powershell
   adb devices
   ```
   Должно показать: `192.168.1.100:5555    device`

## Шаг 2: Подключение с сервера

После настройки на локальном компьютере, подключите устройство с сервера:

### Вариант A: С указанием IP адреса

```bash
/opt/grani/connect_remote_device.sh ВАШ_IP_АДРЕС
```

Например:
```bash
/opt/grani/connect_remote_device.sh 192.168.1.100
```

### Вариант B: Автоматический поиск

```bash
/opt/grani/connect_remote_device.sh --scan
```

Скрипт автоматически просканирует сеть и найдет устройство.

### Вариант C: Прямое подключение через ADB

```bash
export PATH=$PATH:/opt/android-sdk/platform-tools
adb connect ВАШ_IP_АДРЕС:5555
```

## Шаг 3: Использование

После подключения устройства с сервера:

```bash
/opt/grani/dev_run.sh
```

Скрипт автоматически найдет устройство и запустит приложение в режиме разработки.

## Важные замечания

### Требования для работы

1. **Устройство и сервер должны быть в одной сети:**
   - Оба подключены к одной Wi-Fi сети
   - Или настроен VPN/туннель между сетями

2. **Порт 5555 должен быть доступен:**
   - Firewall не должен блокировать порт
   - Если устройство и сервер в разных сетях, нужен проброс порта

### Если устройство и сервер в разных сетях

Если сервер удаленный, а устройство в локальной сети, используйте SSH туннель:

**На локальном компьютере:**
```bash
ssh -R 5555:localhost:5555 user@server
```

**На сервере:**
```bash
adb connect localhost:5555
```

## Проверка подключения

```bash
# На сервере
adb devices

# Должно показать:
# List of devices attached
# 192.168.1.100:5555    device
```

## Отключение

```bash
# Отключить конкретное устройство
adb disconnect 192.168.1.100:5555

# Отключить все устройства
adb disconnect
```

## Решение проблем

### Устройство не видно на локальном компьютере

1. Проверьте USB подключение
2. Убедитесь, что отладка по USB включена
3. Разрешите отладку при появлении запроса
4. Перезапустите ADB: `adb kill-server && adb start-server`

### Не удается подключиться с сервера

1. **Проверьте сеть:**
   ```bash
   ping ВАШ_IP_АДРЕС
   ```

2. **Проверьте порт:**
   ```bash
   telnet ВАШ_IP_АДРЕС 5555
   ```

3. **Убедитесь, что TCP/IP режим включен:**
   - На локальном компьютере: `adb tcpip 5555`

4. **Проверьте firewall:**
   - Убедитесь, что порт 5555 не заблокирован

### Устройство отключается

- Убедитесь, что Wi-Fi не отключается
- Проверьте настройки энергосбережения
- Для Android 11+ используйте встроенную "Беспроводную отладку"

## Альтернатива: Беспроводная отладка (Android 11+)

Если у вас Android 11 или новее, можно использовать встроенную функцию:

1. **На устройстве:**
   - Настройки → Для разработчиков → Беспроводная отладка
   - Включите "Беспроводная отладка"
   - Нажмите "Параметры беспроводной отладки"
   - Запишите IP адрес и порт (например, `192.168.1.100:12345`)

2. **На сервере:**
   ```bash
   /opt/grani/connect_remote_device.sh 192.168.1.100 12345
   ```

Или напрямую:
```bash
adb connect 192.168.1.100:12345
```

## Полезные команды

```bash
# Список всех устройств
adb devices

# Получить IP адрес устройства (если подключено по USB)
adb shell "getprop dhcp.wlan0.ipaddress"

# Перезапустить ADB сервер
adb kill-server && adb start-server

# Проверка подключения
adb get-state
```
