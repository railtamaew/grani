## Root cause

1. **POST /api/vpn/xray/create-client**  
   - Выполняет SSH к VPN-серверу, генерацию конфига Xray, регистрацию клиента.  
   - Длительность 5–6 секунд приводит к ощущению зависания в приложении.

2. **GET /api/vpn/status**  
   - Синхронная проверка статуса через SSH/health-check на серверах.  
   - Задержки до 8+ секунд, при таймауте клиента ~10 с интерфейс «зависает».

## Рекомендации

### Краткосрочные

1. **xray/create-client**  
   - Вынести создание клиента в фоновую задачу (Celery).  
   - Сразу возвращать 202 Accepted и проверять статус через polling или WebSocket.

2. **vpn/status**  
   - Добавить таймаут на проверку (например, 2 с).  
   - По истечении — возвращать кэшированный/последний известный статус или статус «unknown».

3. **Повысить таймауты на мобильном клиенте**  
   - Для этих эндпоинтов временно увеличить до 15–20 с, чтобы уменьшить количество ложных «зависаний».

### Среднесрочные

4. **Redis health check**  
   - Добавить проверку Redis в `/health` для мониторинга.

5. **Кэширование статуса VPN**  
   - Кэшировать результат проверки статуса на 10–30 с.

6. **Асинхронность SSH**  
   - Выполнять SSH-команды асинхронно и не блокировать основной поток.
