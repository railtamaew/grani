# Диагностика проблемы генерации планов в Cursor

## Проблема
Генерация планов задач в Cursor не работает на нескольких устройствах. Раньше планы создавались менее чем за 5 минут, сейчас не создаются вообще.

## Важные выводы

### ✅ Ограничения на вашем сервере НЕ влияют на Cursor
- Ограничения Nginx (таймауты 30s, rate limiting) влияют только на запросы к вашему API (`api.granilink.com`)
- Cursor работает локально и не использует ваш сервер для генерации планов
- Инструмент `todo_write` работает внутри Cursor и не делает HTTP-запросы к вашему серверу

### ⚠️ Возможные причины проблемы

#### 1. Проблемы с API Cursor (наиболее вероятно)
Cursor использует внешний API для генерации планов. Если этот API недоступен или имеет ограничения:
- **Симптомы**: Планы не генерируются на всех устройствах
- **Решение**: Проверить статус сервисов Cursor, возможно временные ограничения API

#### 2. Сетевые ограничения
Если все устройства находятся за одним прокси/firewall:
- **Симптомы**: Проблема на всех устройствах в одной сети
- **Проверка**: 
  ```bash
  # Проверить доступность API Cursor
  curl -I https://api.cursor.sh
  curl -I https://www.cursor.sh
  ```

#### 3. DNS проблемы
Если DNS не может разрешить домены Cursor:
- **Симптомы**: Cursor не может подключиться к своим сервисам
- **Проверка на сервере** (уже выполнена):
  ```bash
  nslookup api.cursor.sh  # Не разрешается
  ```

#### 4. Ограничения провайдера/корпоративной сети
- **Симптомы**: Проблема только в определенной сети
- **Решение**: Проверить с другого интернет-соединения

## Диагностика на вашем сервере

### Проверка DNS
```bash
# DNS не может разрешить api.cursor.sh
nslookup api.cursor.sh
# Результат: "Can't find api.cursor.sh: No answer"
```

### Проверка сетевого доступа
```bash
# Доступ к интернету есть
curl -v https://www.google.com  # Работает

# UFW активен, но разрешает исходящие соединения
sudo ufw status
```

### Выводы
- Сервер имеет доступ к интернету
- DNS на сервере не может разрешить `api.cursor.sh` (но это не критично, если Cursor не использует этот домен)
- Firewall настроен правильно

## Рекомендации по решению

### 1. Проверка на устройствах пользователя

#### Windows
```powershell
# Проверить доступность API Cursor
Test-NetConnection api.cursor.sh -Port 443
Test-NetConnection www.cursor.sh -Port 443

# Проверить DNS
nslookup api.cursor.sh
```

#### Linux/Mac
```bash
# Проверить доступность
curl -I https://api.cursor.sh
curl -I https://www.cursor.sh

# Проверить DNS
nslookup api.cursor.sh
dig api.cursor.sh
```

### 2. Проверка настроек Cursor

#### Проверить настройки прокси в Cursor
1. Открыть настройки Cursor (Ctrl+,)
2. Найти "Proxy" или "Network"
3. Проверить, не установлен ли прокси, который блокирует доступ

#### Проверить логи Cursor
- Windows: `%APPDATA%\Cursor\logs\`
- Mac: `~/Library/Application Support/Cursor/logs/`
- Linux: `~/.config/Cursor/logs/`

Искать ошибки типа:
- `Connection timeout`
- `DNS resolution failed`
- `API request failed`

### 4. Проверка на уровне сети

#### Если устройства в корпоративной сети
- Проверить, не блокирует ли корпоративный firewall/proxy API Cursor
- Обратиться к IT-отделу для проверки логов firewall

#### Если используется VPN
- Попробовать отключить VPN
- Проверить, не блокирует ли VPN API Cursor

### 5. Альтернативные решения

#### Использование локальной генерации планов
Если API Cursor недоступен, можно:
1. Создавать планы вручную
2. Использовать другие инструменты планирования
3. Ожидать восстановления сервиса Cursor
