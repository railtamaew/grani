# Тестирование протоколов и device_id

Этот документ описывает систему тестирования протоколов VPN и механизма device_id **перед сборкой APK**.

## Проблема

Ранее мы исправляли логику протоколов и device_id, собирали APK, тестировали на устройстве, находили ошибки, и цикл повторялся. Это неэффективно.

## Решение

Теперь мы тестируем протоколы и device_id на сервере **до сборки APK**:

1. ✅ Тестируем протоколы (WireGuard, Xray, OpenVPN) без реального подключения
2. ✅ Тестируем механизм device_id (регистрация, логи, 404) через mock API
3. ✅ Проверяем форматы и логику без запуска мобильного приложения
4. ✅ Только после успешных тестов переносим изменения в мобильное приложение

## Быстрый старт

### Все тесты сразу

```bash

python3 scripts/test_all.py
```

#### Тесты протоколов

```bash

python3 scripts/test_protocols.py

# Только WireGuard
python3 scripts/test_protocols.py --protocol wireguard

# Только Xray
python3 scripts/test_protocols.py --protocol xray
```

#### Тесты device_id

```bash

python3 scripts/test_device_id.py

# Только регистрация
python3 scripts/test_device_id.py --scenario register

# Только отправка логов
python3 scripts/test_device_id.py --scenario logs

# Только обработка 404
python3 scripts/test_device_id.py --scenario 404
```

## Что тестируется

### Протоколы

#### 1. API значения протоколов
- ✅ Проверка соответствия значений (`wireguard`, `xray_vless`, и т.д.)
- ✅ Отсутствие пробелов и правильный регистр
- ✅ Соответствие значениям в мобильном приложении

#### 2. Определение протокола
- ✅ VLESS с REALITY определяется правильно
- ✅ VLESS с TLS определяется правильно
- ✅ VMESS определяется по наличию `aid`

#### 3. Форматы конфигураций
- ✅ Xray протоколы (VLESS, VMESS, REALITY)
- ✅ WireGuard формат
- ✅ Структура `supported_protocols` сервера

### Device ID

#### 1. Регистрация устройства
- ✅ Первая регистрация возвращает 200
- ✅ Повторная регистрация возвращает 409
- ✅ Правильная обработка ответов

#### 2. Отправка логов
- ✅ Успешная отправка для зарегистрированного устройства (200)
- ✅ 404 для незарегистрированного устройства
- ✅ Правильный формат логов

#### 3. Механизм повторной регистрации
- ✅ При 404 устройство регистрируется повторно
- ✅ После регистрации логи отправляются успешно
- ✅ Ограничение количества попыток (максимум 3)

#### 4. Формат device_id
- ✅ Валидные форматы (Android, iOS, Web)
- ✅ Отклонение невалидных форматов (пустые, пробелы)

## Mock API

Тесты device_id используют Mock API, который имитирует:
- Регистрацию устройства (`/vpn/device/register`)
- Отправку логов (`/vpn/logs/send`)
- Обработку 404 ошибок
- Механизм повторной регистрации

Это позволяет тестировать логику **без реального сервера**.

## Интеграция в процесс разработки

### Workflow:

```
1. Изменение логики в Python-скриптах
   ↓
2. Тестирование (протоколы + device_id)
   ↓
3. Перенос в мобильное приложение (Dart/Kotlin)
   ↓
4. Сборка APK
   ↓
5. Тестирование на устройстве
```

### Преимущества:

- ⚡ Тестирование за секунды (без сборки APK)
- 🐛 Нахождение ошибок до сборки
- 🔄 Регрессионные тесты для разных сценариев
- 📖 Документированный процесс

## Troubleshooting

### Ошибка: "ModuleNotFoundError: No module named 'test_singbox_config'"

Убедитесь, что вы запускаете скрипты из корня проекта:
```bash
cd /opt/grani
python3 scripts/test_protocols.py
```

### Ошибка в тестах протоколов

Проверьте:
1. Правильность API значений протоколов
2. Логику определения протокола
3. Формат конфигураций

### Ошибка в тестах device_id

Проверьте:
1. Формат device_id
2. Логику регистрации
3. Обработку 404 ошибок
