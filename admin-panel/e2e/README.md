# E2E тесты админ-панели (Playwright)

## Запуск

**Вариант 1 — Playwright сам запускает приложение**

```bash
npm run test:e2e
```

Приложение будет запущено через `npm run start`, после готовности запустятся тесты. Если на порту 3000 уже что-то запущено, будет использовано оно (`reuseExistingServer: true`).

**Вариант 2 — приложение уже запущено**

```bash
SKIP_WEBSERVER=1 npm run test:e2e
```

Либо в другом терминале: `npm start`, затем `npm run test:e2e` (без SKIP_WEBSERVER, тогда поднимется второй процесс, но подключится к уже работающему серверу).

## Тесты с логином

Чтобы прогнать сценарии с авторизацией (редирект на dashboard, вкладки после входа), задайте переменные:

```bash
ADMIN_EMAIL=admin@example.com ADMIN_PASSWORD=secret npm run test:e2e
```

Без них тесты логина/вкладок после входа будут пропущены.

## Отчёт и UI

- Отчёт: `npx playwright show-report` после прогона.
- Режим с интерфейсом: `npm run test:e2e:ui`.

## Структура

- `login.spec.ts` — страница логина (форма, редирект при успешном входе).
- `tabs.spec.ts` — редирект на логин без авторизации; загрузка дашборда, пользователей, серверов, настроек после входа.
