# Тестовые проекты для VPN протоколов

Структура для тестирования VPN протоколов снизу вверх без изменений основного приложения.

## 🎯 Стратегия

**Принцип:** Тестировать протоколы изолированно в минимальном приложении, затем переносить проверенные изменения в основное приложение.

### Преимущества

1. ✅ **Основное приложение не затрагивается** - ничего не сломается
2. ✅ **Изолированное тестирование** - проблемы видны сразу
3. ✅ **Постепенная интеграция** - только проверенные изменения
4. ✅ **Легко откатить** - если что-то не работает

## 🚀 Быстрый старт

### 1. Сборка и установка минимального приложения

```bash
cd /opt/grani/test-projects/vpn-minimal
./scripts/build_and_install.sh
```

### 2. Запуск с логированием

```bash
./scripts/run_with_logs.sh
```

### 3. Тестирование

1. Откройте приложение на устройстве
2. Нажмите "Подключиться"
3. Разрешите VPN подключение (если запросит)
4. Наблюдайте логи
5. Нажмите "Отключиться"

## 📊 План тестирования

### Уровень 1: Парсинг конфигураций ✅

- Парсинг WireGuard конфигурации
- Парсинг Sing-box конфигурации
- Валидация ключей
- Валидация серверов

**Статус:** Готово к тестированию

### Уровень 2: Библиотеки изолированно 🔄

- WireGuard библиотека
- Sing-box библиотека
- XRay библиотека

**Статус:** В разработке

### Уровень 3: Интеграция с Android VPN 🔄

- Создание TUN интерфейса
- Передача конфигурации в библиотеку
- Запуск VPN через библиотеку
- Проверка передачи трафика

**Статус:** В разработке

### Уровень 4: Полный цикл подключения/отключения 🔄

- Подключение через кнопку
- Отключение через кнопку
- Обработка ошибок
- Логирование

**Статус:** В разработке

### Уровень 5: Перенос в основное приложение ⏳

- Интеграция проверенных библиотек
- Обновление основного приложения
- Финальное тестирование

**Статус:** Ожидает успешных тестов уровней 1-4

## ⚠️ Важно

- Все тесты выполняются на **физическом устройстве**
- Основное приложение (`mobile-app/`) **не изменяется** до успешных тестов
- Все изменения документируются и логируются

## 🔄 Процесс работы

1. **Тестирование в минимальном приложении**
   - Создать/обновить тестовый проект
   - Протестировать на физическом устройстве
   - Собрать логи и результаты

2. **Анализ результатов**
   - Что работает
   - Что не работает
   - Известные проблемы

3. **Исправление и повторное тестирование**
   - Исправить проблемы
   - Повторить тесты
   - Убедиться, что все работает

4. **Перенос в основное приложение**
   - Создать ветку в git
   - Перенести проверенный код
   - Протестировать в основном приложении
   - Закоммитить изменения
