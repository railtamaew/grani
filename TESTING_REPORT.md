## ✅ Исправленные проблемы

### 1. API недоступен (502 Bad Gateway) - ИСПРАВЛЕНО

**Проблема была:**
- Nginx пытался подключиться к `http://127.0.0.1:8010` вместо `http://127.0.0.1:8000`
- Бэкенд не был перезапущен после изменений

**Решение:**
1. ✅ Перезапущен бэкенд (убиты старые процессы, запущен новый на порту 8000)
2. ✅ Исправлена конфигурация nginx: изменен порт с 8010 на 8000 в `/etc/nginx/sites-available/granivpn-production`
3. ✅ Перезагружен nginx: `sudo systemctl reload nginx`
4. ✅ API теперь работает: `curl https://admin.granilink.com/api/admin/auth/login` возвращает токен

## 🔧 Выполненные исправления

### 1. Перезапуск бэкенда
```bash
# Остановка всех процессов uvicorn
pkill -f "uvicorn main:app"

# Запуск нового процесса
cd /opt/grani/backend
/usr/bin/python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &
```

### 2. Исправление конфигурации nginx
```bash
# Изменение порта с 8010 на 8000
sudo sed -i 's/127.0.0.1:8010/127.0.0.1:8000/g' /etc/nginx/sites-available/granivpn-production

# Проверка конфигурации
sudo nginx -t

# Перезагрузка nginx
sudo systemctl reload nginx
```

### 3. Проверка работы API
```bash
# Локально
curl -X POST http://127.0.0.1:8000/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"rail.tamaew@gmail.com","password":"ChangeThisPassword123!"}'

# Через nginx
curl -X POST https://admin.granilink.com/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"rail.tamaew@gmail.com","password":"ChangeThisPassword123!"}'
```

## 📝 Примечания

- Учетные данные для тестирования: rail.tamaew@gmail.com / ChangeThisPassword123!
- Пользователь имеет роль `owner` и должен иметь доступ ко всем функциям
- Runtime config настроен на same-origin (`apiBaseUrl: ''`), что означает использование `/api` относительно текущего домена
